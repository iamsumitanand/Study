Information Theory

The mathematics of surprise — and the true meaning of every loss function you've used.

Cross-entropy loss — the loss function behind logistic regression, neural networks, and language models — is a direct measurement from information theory. Understand entropy and every loss function makes sense. Skip it and you're tuning numbers you don't understand.

The Intuition

Information is surprise. Entropy is average surprise.

Shannon's insight: the amount of information carried by an event is inversely proportional to how expected it was. "The sun rose this morning" carries almost no information — you already knew that. "The server crashed during the demo" carries a lot. Mathematically, information is -log p(x): rare events (small p) produce large values, common events produce small ones.

Entropy H(X) is just the weighted average of this surprise over all possible outcomes. It answers: on average, how surprised will I be by a draw from this distribution? A fair coin has entropy of 1 bit — each flip is maximally uncertain. A biased coin that lands heads 99% of the time has entropy near zero — you already know what's coming.

The key insight about uniform distributions: entropy is maximized when all outcomes are equally likely. Knowing nothing about which outcome will occur means every draw is maximally surprising. Any bias — any departure from uniformity — reduces entropy because some outcomes become predictable. A distribution concentrated entirely on one outcome has zero entropy: it holds no information because it holds no surprise.

  • A rare event (low p) carries high information when it occurs.
  • A common event (high p) carries low information — you expected it.
  • Entropy measures the average information content of a distribution.
  • Cross-entropy measures average information when you encode outcomes using the wrong distribution.

Understanding It Deeply

Build the intuitions from first principles

Three questions that every ML practitioner should be able to answer. Each one unlocks a different layer of why the math is the way it is.

Think first Why is -log p(x) the natural measure of information in an event — and not, say, 1/p(x) or 1 - p(x)?

Two properties uniquely determine the form. First, information must be additive for independent events: if I flip two independent coins, the total information should be the sum of each flip's information. This rules out 1/p(x) and 1-p(x), which don't satisfy additivity for independent events (where p(x,y) = p(x)·p(y)). Second, information must be monotonically decreasing in probability — more likely events are less surprising. The logarithm is the unique function satisfying both: -log(p·q) = -log(p) + (-log(q)). The base of the logarithm sets the unit: base 2 gives bits, base e gives nats. ML uses natural log by convention.

Think first Why does minimizing cross-entropy H(P, Q) make the model's distribution Q match the true distribution P?

Cross-entropy is H(P, Q) = -Σ P(x) log Q(x). During training, the true labels define P (one-hot: the correct class has probability 1, all others 0). The model outputs define Q. Cross-entropy measures: how many bits are needed to encode events drawn from P, using a code optimized for Q? This is minimized when Q = P — any divergence wastes bits. But notice the minimum equals H(P) (the entropy of the true labels), and the gap above that minimum is exactly KL divergence: H(P,Q) = H(P) + KL(P||Q). Since H(P) is fixed by the data, minimizing cross-entropy is equivalent to minimizing KL divergence — pushing the model's distribution toward the true one.

Think first KL divergence is asymmetric: KL(P||Q) ≠ KL(Q||P). Why does this asymmetry matter in practice?

The two directions penalize different failure modes. KL(P||Q) — "forward KL" — measures how well Q covers P. Wherever P(x) > 0 and Q(x) ≈ 0, the term P(x)·log(P(x)/Q(x)) blows up to infinity. So minimizing forward KL forces Q to put mass everywhere P does — it is mass-covering. When P has multiple modes, Q will spread across all of them, producing a blurry average. This is what cross-entropy loss does in classification — it forces the model to account for all true classes.

KL(Q||P) — "reverse KL" — diverges when P(x) ≈ 0 and Q(x) > 0. Minimizing it forces Q to avoid regions where P has no mass, making Q mode-seeking: it picks one mode of P and concentrates there. This is the direction used in variational inference and many generative models — which explains why VAEs and diffusion models tend to produce sharp, committed samples rather than blurry averages.

The Math

The core equations

Five formulas that the entire field traces back to. Each one builds on the last.

Self-Information
I(x) = −log p(x)
p(x) probability of event x — how likely it was before it happened log natural logarithm (base e) in ML; base 2 gives bits − negation so that rare events (small p) yield large positive information
Entropy
H(X) = −Σ p(x) log p(x) = 𝔼[I(x)]
Σ sum over all possible outcomes x p(x) log p(x) probability-weighted surprise — contribution of each outcome 𝔼[I(x)] expected self-information — average surprise over the distribution
Cross-Entropy
H(P, Q) = −Σ P(x) log Q(x)
P(x) true distribution — the actual probabilities (e.g. one-hot labels) Q(x) model distribution — what your model predicts H(P,Q) average bits needed to encode P-events using a code built for Q
KL Divergence
KL(P‖Q) = Σ P(x) log(P(x) / Q(x)) = H(P,Q) − H(P)
KL(P‖Q) extra bits wasted by using Q instead of the true distribution P H(P,Q) − H(P) cross-entropy minus entropy = the inefficiency penalty alone ≥ 0 always non-negative (Gibbs' inequality); equals 0 iff P = Q everywhere
Mutual Information
I(X; Y) = H(X) − H(X | Y) = KL(P(X,Y) ‖ P(X)·P(Y))
H(X|Y) conditional entropy — uncertainty in X after observing Y H(X) − H(X|Y) reduction in uncertainty about X that knowing Y provides I(X;Y) = 0 iff X and Y are independent — knowing Y tells you nothing about X

Worked Example: 4-class cross-entropy loss

A classifier outputs probabilities for four classes: [0.70, 0.20, 0.07, 0.03]. The true label is class 1 (the first class). Compute the cross-entropy loss.

1 Write out the true distribution P (one-hot for class 1):
1 P = [1, 0, 0, 0]    Q = [0.70, 0.20, 0.07, 0.03]
2 Apply H(P,Q) = −Σ P(x) log Q(x). Only the term where P(x) = 1 survives:
2 H(P,Q) = −1·log(0.70) − 0·log(0.20) − 0·log(0.07) − 0·log(0.03)
3 Compute the single surviving term:
3 H(P,Q) = −log(0.70) = 0.3567 nats
Interpretation: the loss is 0.3567. If the model had assigned probability 1.0 to the correct class, loss would be 0. If it had assigned 0.01, loss would be 4.6 — a 13× penalty. With one-hot labels, cross-entropy loss always reduces to −log Q(correct class). The model is penalized logarithmically for underestimating the true class. This is why a model that says 0.01 when the answer is class 1 is catastrophically penalized, while 0.90 vs 0.99 is a much smaller gap.

From Theory to Real ML

Information theory shows up everywhere

These aren't analogies — they are direct applications of the same equations you just learned.

Cross-entropy loss in classification

Every softmax classifier minimizes H(P,Q) where P is the one-hot true label and Q is the model output. Because P is one-hot, the formula collapses to −log Q(true class) — the negative log probability assigned to the correct answer. The model is pushed to concentrate probability mass on the right class. Logistic regression is a 2-class special case.

KL divergence in the VAE ELBO

The Variational Autoencoder loss is L = reconstruction loss + KL(q(z|x) ‖ p(z)). The KL term regularizes the learned encoder distribution q(z|x) toward a standard Gaussian prior p(z). Without it, the encoder could collapse to a degenerate point mass for each input — perfect reconstruction, but no usable latent space. The KL penalty forces the latent codes to remain spread out and continuous, making interpolation and sampling from the latent space meaningful.

Perplexity in language models

Perplexity is exp(H(P,Q)) — the exponential of the cross-entropy. A perplexity of 20 means the model is as surprised as if it were choosing uniformly among 20 equally likely next tokens. Lower is better. The key property: perplexity is in token space, making it interpretable regardless of vocabulary size. A language model with perplexity 10 has learned to narrow down the next word to roughly 10 candidates on average.

Mutual information in feature selection

I(feature; label) measures how much knowing a feature reduces uncertainty about the label. Features with high mutual information carry genuine signal; low MI features are noise. Unlike correlation, MI captures non-linear relationships. Decision tree splits (information gain) are literally computed as the reduction in entropy H(Y) - H(Y|feature) — the mutual information between the feature and the label.

The unifying thread: Every one of these uses the same core move — measuring uncertainty, then measuring how much of that uncertainty a model, feature, or encoding eliminates. That's information theory: a calculus for quantifying what you know and what you don't.

Common Misconceptions

Five wrong beliefs that block real understanding

These are the misconceptions that cause real confusion when reading papers and debugging models.

  • ① Entropy = disorder (like thermodynamic entropy)
    Shannon entropy and thermodynamic entropy share a formula and a name, but they measure fundamentally different things. Shannon entropy measures uncertainty in a probability distribution — it's purely a property of information. Thermodynamic entropy measures physical disorder of a macroscopic system. The connection is deep (Boltzmann's S = k log W is indeed an information-theoretic argument), but treating them as interchangeable generates false intuitions. In ML, entropy always means Shannon entropy — average surprise over a distribution.
  • ② Cross-entropy is cross-correlation with a negative sign
    They share the word "cross" and nothing else. Cross-correlation is a signal-processing operation measuring similarity between two signals as a function of lag. Cross-entropy H(P,Q) is an information-theoretic quantity measuring the expected log-loss when using model distribution Q to encode samples from true distribution P. No time-domain structure, no convolution, no signals. The terminology collision is unfortunate but purely historical.
  • ③ Lower cross-entropy loss means the model is better calibrated
    Lower cross-entropy means the model assigns higher probability to correct classes on the training set. Calibration is a separate question: does a 70% confidence prediction actually correspond to 70% accuracy? A model can have very low cross-entropy loss (excellent at concentrating probability on the right class) while being catastrophically miscalibrated (its confidence scores don't match empirical accuracy). Calibration requires separate measurement — Expected Calibration Error (ECE), reliability diagrams — not just reading the loss.
  • ④ KL divergence is a distance measure — it satisfies the triangle inequality
    KL divergence is not a metric. It violates two metric axioms: it is not symmetric (KL(P‖Q) ≠ KL(Q‖P) in general), and it does not satisfy the triangle inequality. The correct term is "divergence" — a directed measure of dissimilarity. When a symmetric distance is needed, the Jensen-Shannon divergence JSD(P‖Q) = (KL(P‖M) + KL(Q‖M))/2 where M = (P+Q)/2 is used. JSD is symmetric, bounded in [0,1], and its square root is a true metric. GANs use JSD implicitly.
  • ⑤ To use mutual information, you need to know the full joint distribution
    In theory, I(X;Y) = KL(P(X,Y) ‖ P(X)P(Y)) requires knowing joint and marginal distributions exactly. In practice, MI can be estimated from samples without knowing the distribution: kernel density estimation, binning, or modern neural estimators like MINE (Mutual Information Neural Estimation) approximate MI from data. Decision trees estimate it via empirical frequencies. The theoretical definition requires distributions; practical computation requires only samples — which is why MI is usable in high-dimensional ML settings.

Summary

Five things to lock in

  • Self-information: I(x) = -log p(x). Rare events carry large information; common events carry small. The log is the only function that makes information additive for independent events.
  • Entropy H(X) is average surprise. Maximum at uniform distributions. Zero when one outcome has probability 1. Measures irreducible uncertainty in a distribution.
  • Cross-entropy H(P,Q) = H(P) + KL(P‖Q). Minimizing cross-entropy is minimizing KL divergence from model to truth. With one-hot labels this collapses to -log Q(correct class).
  • KL divergence is not a distance: it is asymmetric and violates the triangle inequality. Forward KL is mass-covering (used in cross-entropy loss); reverse KL is mode-seeking (used in variational inference).
  • Every major ML loss — classification cross-entropy, VAE ELBO, language model perplexity, decision tree splits — is a direct application of these five equations. The math is not analogous; it is identical.
Now the math is in place. Time to understand how models actually train. You can now explain why cross-entropy is the right loss function, what minimizing it actually means, and why KL divergence appears everywhere from VAEs to RLHF. The formulas that were mysterious are now derivable. The next question: given a loss function, how does a model with millions of parameters efficiently find its minimum? Test Your Understanding

9 Questions

0 / 9 revealed

01 A fair 8-sided die is rolled. How much self-information (in bits) does learning the outcome carry? What if one face had probability 1? ▼
3 bits; 0 bits. For a fair die: I(x) = -log₂(1/8) = log₂(8) = 3 bits. Each face is equally likely, so every outcome is maximally surprising — 3 bits of information. If one face has probability 1: I(x) = -log₂(1) = 0. The outcome was certain; there is nothing to learn. This is the clearest illustration of why information measures surprise: certainty gives zero information.
02 Why does entropy decrease when a uniform distribution is made non-uniform? Give the intuition and point to what changes in the formula. ▼
Because some outcomes become predictable — they no longer contribute surprise. In H(X) = -Σ p(x) log p(x), each term is p(x) · I(x). When p increases toward 1 for one outcome, its self-information I(x) → 0 (certain events carry no information). Meanwhile, the other outcomes' probabilities shrink, so they also contribute less. The entropy-maximizing distribution is uniform because it maximally spreads the uncertainty — no outcome can be confidently predicted. Any deviation concentrates probability, making the distribution more predictable and reducing average surprise.
03 A 3-class classifier outputs [0.60, 0.30, 0.10]. The true class is class 2 (the second one). Compute the cross-entropy loss. ▼
1.204 nats. With one-hot labels: H(P,Q) = -log Q(true class) = -log(0.30) = 1.204 nats. If the model had correctly assigned the highest probability to class 2 (say 0.70), the loss would be -log(0.70) = 0.357 — a 3.4× reduction. The model is not wrong, but it is uncertain — and cross-entropy directly penalizes that uncertainty about the correct answer.
04 KL divergence KL(P‖Q) = 0 for some P ≠ Q. Is this possible? Why or why not? ▼
No — KL(P‖Q) = 0 if and only if P = Q everywhere. This is Gibbs' inequality. KL(P‖Q) = Σ P(x) log(P(x)/Q(x)) = -Σ P(x) log(Q(x)/P(x)). By Jensen's inequality applied to the concave log function: -log(𝔼[Q/P]) ≤ 𝔼[-log(Q/P)] = KL. Since 𝔼[Q/P] = Σ P·(Q/P) = 1, we get KL ≥ -log(1) = 0. Equality holds iff Q/P is constant everywhere P > 0, which forces Q = P. This is why KL is used to measure distributional similarity — it is zero iff the distributions are identical.
05 A language model has perplexity 100. What does this mean in concrete terms? Is lower perplexity always better? ▼
The model is as surprised as if choosing uniformly among 100 equally likely next tokens. Perplexity = exp(cross-entropy loss). Perplexity 100 means each token prediction carries the same uncertainty as a uniform choice among 100 tokens. Lower perplexity means the model is more confident and accurate about what comes next — yes, lower is better for the task the model was trained on. However: perplexity is task-specific (a model with low perplexity on Wikipedia may have high perplexity on legal documents), and it measures fit to the training/evaluation distribution, not downstream task quality. A model with lower perplexity can still perform worse on summarization, reasoning, or instruction following.
06 In a VAE, what happens if you remove the KL term from the ELBO loss and train with reconstruction loss alone? ▼
The encoder collapses to a deterministic function — the model becomes a regular autoencoder. Without the KL penalty, the optimal encoder learns to map each input to a distinct, tight cluster in latent space (near-zero variance Gaussian). This minimizes reconstruction loss perfectly but makes the latent space discontinuous — you cannot interpolate between points or sample new images by drawing from the prior, because most of the latent space maps to meaningless decoder outputs. The KL term is not regularization in the loose sense — it is the mechanism that makes the latent space continuous and the prior p(z) actually match the aggregate posterior.
07 Two features are being evaluated for a decision tree split. Feature A has MI = 0.5 nats with the label. Feature B has correlation 0.9 with the label. Which should you prefer — and can you tell? ▼
Feature A — but the comparison is not apples-to-apples. Mutual information and correlation measure different things. MI captures all statistical dependence (linear and non-linear); correlation only measures linear dependence. A feature with correlation 0.9 may have high MI, but a feature with correlation 0.3 could also have MI = 0.8 if the relationship is non-linear. To compare them properly, you need MI for both. For a decision tree, information gain (which is MI) is the correct criterion — it directly measures how much the feature reduces label uncertainty. High correlation is evidence of a useful feature but is not the same as high MI.
08 A model achieves 95% test accuracy but its cross-entropy loss is still high. How is this possible and what does it indicate? ▼
The model is predicting the right class but with low confidence — it is under-confident and possibly miscalibrated. Cross-entropy = -log Q(correct class) with one-hot labels. If the model says [0.35, 0.34, 0.31] and the answer is class 1, it predicts correctly (argmax = class 1) but the loss is -log(0.35) = 1.05 — high, because the model hedges. Accuracy only measures whether argmax is right; cross-entropy penalizes diffuse probability mass. This pattern signals miscalibration: the model is not committing to answers it is effectively right about. Temperature scaling can often fix this by sharpening the output distribution post-training.
09 Why is cross-entropy loss preferred over mean squared error for classification tasks? Give the information-theoretic explanation. ▼
Because cross-entropy is the correct loss for maximum likelihood estimation of a categorical distribution, and MSE is not. Maximizing the log-likelihood of a softmax model under one-hot labels gives exactly the cross-entropy objective — it is derived from the definition of the categorical distribution, not imposed by convention. MSE minimization corresponds to maximum likelihood under a Gaussian noise model, which is appropriate for real-valued regression but not for probability outputs. Practically: MSE gradients vanish when the sigmoid/softmax output saturates (e.g., predicts 0.999 for the wrong class), while cross-entropy gradients remain proportional to the prediction error — ∂L/∂z = Q - P. This makes cross-entropy train faster and more reliably for classification.
