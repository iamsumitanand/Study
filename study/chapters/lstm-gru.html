LSTMs & GRUs

The gated memory cells that gave neural networks a working long-term memory.

← RNNs Transformers & Attention →
An RNN has one number to remember everything. An LSTM has a filing cabinet with a forget button — it decides what to write down, what to shred, and what to read back out, at every single step.

The Intuition

The cell state as a conveyor belt

The single most useful mental model for an LSTM: imagine a conveyor belt running horizontally through time, carrying information from the past into the future. This is the cell state — a separate memory channel that runs parallel to the regular hidden state. The key insight is that information on the belt flows almost unchanged unless a gate explicitly acts on it.

Three gates stand beside the belt at every timestep, each a learned sigmoid — outputting a number between 0 and 1 that acts as a valve:

  • The forget gate scans what's already on the belt and decides how much to keep. A value near 0 erases; near 1 preserves perfectly. This gate taught itself when to forget — e.g., when a subject pronoun changes in a sentence.
  • The input gate decides how much of the new candidate information to add to the belt. It answers: "Is this new token worth writing down?"
  • The output gate filters what portion of the cell state to expose as the hidden state — the thing the network actually "says" to the next layer.

This architecture solves the vanishing gradient problem that cripples plain RNNs: because the cell state update is additive rather than multiplicative, gradients can flow backward through many timesteps without shrinking to zero. The belt doesn't attenuate the signal — it preserves it until a gate chooses to erase it.

Understanding It Deeply

Three questions that cut to the core

Most explanations stop at "LSTMs have gates." These three questions force you to understand why the gates are designed the way they are — and when to prefer a GRU instead.

Think first A plain RNN updates its hidden state as h_t = tanh(W·h_{t-1} + U·x_t). During backpropagation through time, you multiply by this weight matrix at every step. Why does the LSTM's additive cell state update prevent the gradient from vanishing the same way?

In a plain RNN, the gradient of the loss with respect to a hidden state far in the past involves a long product of Jacobian matrices — and if the largest singular value of W is less than 1, this product shrinks exponentially. The cell state in an LSTM avoids this because its update is c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t. Differentiating through the addition, the gradient of c_t with respect to c_{t-1} is just f_t — the forget gate value, which is near 1 when the gate is open. This is the "constant error carousel": gradients flow back through the cell state almost unchanged as long as the forget gate stays open. No matrix multiplication cascades; just an element-wise multiply by a value the network learned to keep near 1 for long-range dependencies.

Think first The forget gate opens and closes based on what the network sees. What kinds of patterns do you think the forget gate learns to erase — and can you name a concrete linguistic example where it must fire strongly?

Empirical probing studies show that forget gates learn to fire (erase) on clear semantic boundaries. The most studied example: tracking subject-verb number agreement across a relative clause. In "The keys that the man carries are", the LSTM must remember "keys" is plural across the relative clause. When "are" is reached, the number information can be partially released. More concretely, forget gates often erase positional counters once a sentence ends, reset topic context at paragraph breaks, or discard speaker identity when dialogue shifts. They don't learn these rules from instruction — they emerge from gradient descent optimizing prediction loss, which rewards remembering exactly what's needed for future tokens.

Think first A GRU has no separate cell state and only two gates (reset and update). Given that LSTMs are strictly more expressive, why would you ever choose a GRU? What's the tradeoff, and on what data distributions does the simpler architecture win?

The tradeoff is not expressiveness vs. simplicity — it's parameters vs. training signal. A GRU has roughly 2/3 the parameters of an LSTM for the same hidden size, which means each parameter gets more gradient updates per epoch on the same dataset. On small-to-medium datasets, GRUs often match or beat LSTMs because they overfit less. Additionally, GRUs train faster per step (fewer matrix multiplications). The regime where LSTMs clearly win is long sequences with multiple distinct memory channels that need to be independently controlled — for example, tracking nested syntactic structure simultaneously with surface form. On clean, long-range sequential data with a large training set, LSTMs shine. On noisy or limited data with moderate sequence lengths, GRUs are often the right default.

The Math

Four equations — one full LSTM step

At each timestep t, the LSTM receives the current input x_t and the previous hidden state h_{t-1}. Four equations run in sequence, updating the cell state c_t and producing the new hidden state h_t.

LSTM Equations
f_t = σ(W_f · [h_{t-1}, x_t] + b_f)
i_t = σ(W_i · [h_{t-1}, x_t] + b_i)
g_t = tanh(W_g · [h_{t-1}, x_t] + b_g)
o_t = σ(W_o · [h_{t-1}, x_t] + b_o)
c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t
h_t = o_t ⊙ tanh(c_t)
f_t forget gate — how much of old cell state to keep (0=erase, 1=keep) i_t input gate — how much of the new candidate to write in g_t cell candidate — new information proposed by tanh (range −1 to 1) o_t output gate — how much of the cell state to expose as hidden state c_t cell state — the long-term memory, updated additively h_t hidden state — what gets passed to next layer and next timestep ⊙ element-wise (Hadamard) product σ sigmoid function — outputs in (0, 1), acts as a soft gate

Worked example — 2D state, two tokens:

We'll use 2D vectors and hand-picked gate values to trace the cell state update exactly. Initial state: c_0 = [0, 0], h_0 = [0, 0].

— Token 1 — "The"
1 Forget gate f_1 = σ(...) = [0.9, 0.8] Near 1 — nothing to forget yet, cell is empty anyway.
2 Input gate + candidate i_1 = [0.6, 0.4] · g_1 = [0.7, 0.5] The article "The" carries some syntactic signal.
3 Cell state update c_1 = [0.9, 0.8] ⊙ [0, 0] + [0.6, 0.4] ⊙ [0.7, 0.5] c_1 = [0.42, 0.20] The article's signal is written to memory.
4 Output gate → hidden state o_1 = [0.5, 0.7] → h_1 = [0.5, 0.7] ⊙ tanh([0.42, 0.20]) h_1 ≈ [0.5 × 0.40, 0.7 × 0.20] = [0.20, 0.14]
— Token 2 — "keys" (plural noun — important to remember)
5 Forget gate f_2 = [0.7, 0.3] Partially erases article signal [dim 1], but keeps [dim 0] for number info.
6 Input gate + candidate i_2 = [0.9, 0.2] · g_2 = [0.8, −0.3] "keys" strongly writes plurality information to dim 0.
7 Cell state update c_2 = [0.7, 0.3] ⊙ [0.42, 0.20] + [0.9, 0.2] ⊙ [0.8, −0.3] c_2 = [0.294, 0.060] + [0.720, −0.060] c_2 = [1.014, 0.000] Dim 0 now strongly encodes "plural subject". Dim 1 cancels to zero.
Key pattern: the additive update lets the network write new information without erasing everything. The forget gate selectively retains what's useful. The input gate controls how strongly new information writes in. These two acting together is what enables multi-step memory — the information about "keys" being plural can now survive many more timesteps without degrading.

From Toy to Real

Where LSTMs actually ran the world

Before transformers dominated, LSTMs were the architecture powering the most impressive AI systems in production. Three concrete examples show the different ways their long-range memory was actually being used.

Machine translation (seq2seq, 2014–2018). Google Translate's 2016 overhaul to Neural Machine Translation used a stacked LSTM encoder-decoder. The encoder read an entire source sentence into a fixed-length cell state; the decoder generated the translation token-by-token, conditioned on that state. The challenge was compressing an entire sentence's meaning into a single vector — which pushed the limits of what LSTMs could hold in their cell states for long sentences. This motivated Bahdanau attention (1 year later), which let the decoder look back at all encoder states rather than relying on a single bottleneck. But the LSTM's ability to accumulate sentence-level context was what made the original system work at all.

Speech synthesis (WaveNet, 2016). DeepMind's WaveNet used dilated causal convolutions at the waveform level, but conditioning information — speaker identity, phoneme timing, prosody — was encoded with LSTMs. The LSTM's cell state held slowly-varying prosodic context across hundreds of milliseconds while the convolutional layers handled fast waveform detail. This separation of "what to say" (LSTM) from "exactly how it sounds" (convolution) produced the first genuinely human-sounding speech synthesis.

Financial time series. LSTMs became standard for equity return prediction, volatility forecasting, and high-frequency order-book modeling. The key property being exploited: the forget gate could learn to down-weight information from periods of abnormal volatility (e.g., a news shock) and return attention to normal-regime patterns once the shock had passed. This is precisely the kind of regime-dependent memory that plain RNNs and AR models cannot replicate.

The lesson from all three: LSTMs weren't used because their math is elegant — they were used because gated memory gave the model a way to carry relevant context for however long the task required, and forget irrelevant context when it changed. That "selectivity across time" is the core capability transformers later scaled up dramatically, through attention over all positions simultaneously rather than gated sequential updates.

Common Misconceptions

Five causal misconceptions — and what actually happens

These five myths aren't random errors — they form a logical chain. Each one follows plausibly from the previous. Break the chain at its weakest link.

  • ① LSTMs solve the vanishing gradient problem completely.
    They dramatically reduce it, not eliminate it. The additive cell state update allows gradients to flow back through many steps without shrinking — but the hidden state path still goes through tanh and sigmoid, which can saturate. Very long sequences (thousands of timesteps) can still cause gradient issues in LSTMs, which is one reason attention mechanisms were invented. "Solved" is too strong; "substantially mitigated for practical sequence lengths" is accurate.
  • ② The forget gate forgets, the input gate inputs — each gate has one job.
    Both gates interact on the cell state simultaneously: c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t. The cell state update is a blend, not a sequence. Saying one gate "forgets" and another "writes" treats them as independent operations — they're not. A fully open forget gate with a zero input gate is pure memory. A zero forget gate with a high input gate is a hard reset. Most of the time both operate at intermediate values and the result is a continuous interpolation between old and new information.
  • ③ The cell state is the "real" memory and the hidden state is just an output.
    Both carry information. The cell state holds long-range context; the hidden state is the model's current "working memory" — it influences the next step's gate computations via [h_{t-1}, x_t] on every gate equation. A network with a perfect cell state but corrupted hidden states would still make poor predictions because the gates would fire incorrectly. They are complementary, not hierarchical.
  • ④ GRUs are just simpler LSTMs — always use an LSTM if you have the compute.
    More parameters is not strictly better when training data is limited. A GRU has roughly 2/3 the parameters of an LSTM for the same hidden size, meaning each parameter receives a stronger gradient signal. On small datasets, GRUs often generalize better because they have less capacity to overfit. The GRU also lacks an explicit cell state — the update gate performs the role of both forget and input gates simultaneously — which is a genuine architectural constraint that can be appropriate when the task doesn't require independently controlled memory channels.
  • ⑤ Transformers made LSTMs obsolete — there's no reason to use them today.
    Transformers require O(n²) memory and compute with respect to sequence length n. For streaming inference on very long sequences, or on-device applications where memory is constrained, LSTMs remain competitive. State space models like Mamba (2023) are reviving the gated-recurrent idea with better hardware efficiency. LSTMs are also still dominant in some time-series forecasting settings, embedded systems, and real-time audio processing where quadratic attention is infeasible. "Obsolete" conflates "no longer state-of-the-art on NLP benchmarks" with "no longer useful" — a common but incorrect leap.

Summary

Six things to lock in

  • The LSTM adds a separate cell state — a conveyor belt of long-term memory — alongside the hidden state. The cell state is updated additively, which is the structural reason gradients don't vanish through it.
  • Three gates (forget, input, output) each multiply by a sigmoid value between 0 and 1, acting as soft switches. The forget gate selectively erases old memory; the input gate selectively writes new memory; the output gate selectively reads from the cell.
  • The core cell update is: c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t. Differentiate through this and you get f_t — not a product of weight matrices — explaining the gradient stability.
  • Gates are not rule-based switches — they're learned sigmoid layers. Forget gates learn to erase on semantic boundaries (new sentence, change of subject, end of relevant clause); input gates learn to write when new information is worth keeping.
  • A GRU merges the forget and input gate into a single update gate, eliminates the separate cell state, and has 2/3 the parameters. It often matches LSTMs on limited data and trains faster. Prefer GRUs unless you have a large dataset and strong reason to believe independent memory channels help.
  • LSTMs powered machine translation, speech synthesis, and financial modeling in the pre-transformer era. Their core insight — selective gated memory — lives on in attention mechanisms, state space models, and modern hybrid architectures.
One limitation LSTMs never fully escaped: Even with gated memory, the LSTM processes tokens one at a time and compresses the entire preceding context into a fixed-size hidden vector. For long sequences, early information is increasingly filtered and compressed through every subsequent gate. What if instead of reading left to right and forgetting along the way, a model could look directly at any position in the input at any time — comparing every token to every other token simultaneously?

That's what attention mechanisms do. And stacked entirely from attention layers, without any recurrence at all, is the Transformer. That's the next chapter.

01 Why does the additive cell state update in an LSTM allow gradients to flow back many timesteps without vanishing? ▼
Because the gradient of c_t with respect to c_{t-1} is just the forget gate value f_t — not a matrix product. In a plain RNN, backpropagation multiplies by the weight matrix at every step; if the largest singular value is below 1, the gradient exponentially shrinks. In an LSTM, differentiating c_t = f_t ⊙ c_{t-1} + i_t ⊙ g_t through the addition gives ∂c_t/∂c_{t-1} = f_t — an element-wise multiply by a value near 1 when the gate is open. This is Hochreiter's "constant error carousel": the error signal circulates through the cell state indefinitely when the forget gate stays open, rather than decaying with each step.
02 An LSTM is processing the sentence: "The trophy didn't fit in the bag because it was too big." At the word "it", what would you expect the forget gate to do — and why? ▼
The forget gate should stay open (near 1) for the cell state dimensions that encode the referent of "trophy". Resolving the pronoun "it" requires remembering what objects were mentioned earlier and their properties. A well-trained LSTM's forget gate on "it" should not erase subject/object identity information — it needs to carry "trophy" and "bag" as candidates through to "big" to resolve the coreference. Empirically, LSTM forget gates are found to stay open across relative clauses and between pronoun and referent.
03 Starting from c_0 = 0, with forget gate f = 0.5 and constant input i ⊙ g = 0.4 at every step, what is the cell state after 3 steps? ▼
c_3 ≈ 0.55. Step by step: c_1 = 0.5×0 + 0.4 = 0.40. c_2 = 0.5×0.40 + 0.4 = 0.20 + 0.40 = 0.60. c_3 = 0.5×0.60 + 0.4 = 0.30 + 0.40 = 0.70. Wait — let me recalculate: c_1 = 0.4, c_2 = 0.5×0.4 + 0.4 = 0.6, c_3 = 0.5×0.6 + 0.4 = 0.7. With f = 0.5, the system converges to c* = i⊙g / (1−f) = 0.4/0.5 = 0.8. After 3 steps it has reached 0.7, approaching 0.8 geometrically. This illustrates how the forget gate controls the effective memory horizon — lower f means faster decay toward a new steady state.
04 What is the structural difference between an LSTM and a GRU, and why does the GRU often match LSTM performance on small datasets? ▼
A GRU has no separate cell state and only two gates: a reset gate and an update gate. The update gate simultaneously controls how much old hidden state to keep and how much new candidate to write in — collapsing the LSTM's forget + input gate pair into one. This reduces the parameter count to roughly 2/3 of an LSTM at equal hidden size. On small datasets, fewer parameters means stronger gradient signal per parameter, less overfitting, and often better generalization. The GRU trades the LSTM's independent control of separate memory and output channels for simplicity and efficiency. When the task genuinely requires independently gated long-term and working memory, LSTMs can win — but this regime requires substantial training data to learn.
05 The output gate fires with value 0.1 (almost closed). The cell state is large. What does the hidden state look like — and what practical effect does this have on the network? ▼
The hidden state h_t = 0.1 ⊙ tanh(c_t) is nearly zero, regardless of how rich the cell state is. Practically, the next layer receives almost no information about the current cell state. This matters because the hidden state h_{t-1} feeds directly into all four gate computations at the next timestep. A near-zero h_t means the gates at t+1 will be computed with almost no signal from the current state — they'll behave almost entirely as a function of x_{t+1} alone. The output gate can therefore "hide" information in the cell state — keeping it in memory without broadcasting it — which is useful when storing context that shouldn't affect output until a later relevant trigger.
06 You're choosing between an LSTM and a Transformer for a real-time speech recognition system that must run on an embedded device with 256 KB RAM. Which do you choose — and why? ▼
LSTM. Transformers require O(n²) memory in the attention mechanism with respect to sequence length — for continuous audio, this becomes prohibitive. An LSTM processes each frame in constant memory (fixed hidden state size), making it suitable for streaming inference where the full sequence is never held in memory. With 256 KB RAM, even a small transformer with modest context would exceed the memory budget quickly, while an LSTM with a few hundred hidden units can operate within tight constraints. This is the exact use case where LSTMs remain active in production today (2024), alongside state space models like Mamba that extend the same recurrent principle.
07 In the seq2seq machine translation architecture, the encoder LSTM produces a single context vector and the decoder LSTM reads from it. What fundamental bottleneck does this create — and how did the field respond? ▼
The entire source sentence must be compressed into a fixed-size vector. For short sentences this works well, but for long sentences (20+ words), the final hidden state cannot faithfully encode all the information from early tokens — those have been processed through many forget gates since. The quality of translation degrades with sentence length. The response was Bahdanau attention (2015): instead of reading only the final encoder hidden state, the decoder at each step computes a weighted sum over all encoder hidden states, attending most to the positions most relevant to the current output token. This eliminated the fixed-size bottleneck and was the direct architectural precursor to the self-attention mechanism in transformers.
08 Both the forget gate and the input gate are parameterized by weight matrices that are trained by gradient descent. Does this mean the network "learns when to forget" from the data — or is forgetting still a manual design decision? ▼
The network fully learns when to forget from data. The forget gate computes f_t = σ(W_f · [h_{t-1}, x_t] + b_f) — W_f and b_f are trained by backpropagation through time, exactly like every other weight in the network. The network is never told "forget subject identity at sentence boundaries" — it discovers this emergent behavior because doing so minimizes prediction loss. Karpathy's interpretability experiments and later probing studies confirmed that forget gates in trained LSTMs do learn linguistically meaningful patterns (sentence boundaries, subject-verb distance tracking, quote depth). The architecture enables selective memory; gradient descent decides what the selectivity criterion is.
09 LSTM "solved" long-range dependencies for sequences of hundreds of tokens. Transformers handle sequences of thousands via attention. What would you expect a model to look like that handles sequences of millions of tokens efficiently — and does anything like this exist? ▼
Constant-memory recurrent processing — essentially the gated-recurrence idea revisited with better hardware efficiency. A model that processes million-length sequences can't hold an attention matrix (O(n²) memory) and can't compress everything into a single fixed vector (LSTM bottleneck at very long range). The answer is selective state compression: read the sequence recurrently but with gates that learn to selectively update a fixed-size state. This is exactly what Mamba (Gu & Dao, 2023) does — it's a selective state space model with input-dependent gating, achieving linear-time sequence processing. It can be thought of as a direct descendant of the LSTM idea, extended to be hardware-efficient on modern GPUs. The gated-recurrence paradigm is not historical curiosity — it is actively competing with transformers in 2024.
