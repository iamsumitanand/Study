Recurrent Neural Networks

The architecture that gave machines a sense of time — and showed us exactly how hard memory is.

An MLP reads your sentence like a list of words. An RNN reads it like a person — building context word by word. The difference isn't architectural complexity. It's a single loop: the output from step t feeds back in at step t+1. That loop is what gives a network a past.

The Intuition

Memory through time

An MLP takes a fixed-size input vector and maps it to an output. Feed it the word "bank" and it gives you an answer — but it has no idea whether the previous word was "river" or "investment." Sequential data defeats fixed-size architectures because meaning depends on order, not just presence.

An RNN solves this with a hidden state — a vector that persists between time steps. At each step, the network reads the current input and the previous hidden state together, then produces a new hidden state. That hidden state is the RNN's running summary of everything it has seen so far. It's not stored in external memory — it's compressed, lossy, and entirely learned.

  • The input at each step is a single element of the sequence — a word, a frame, a reading.
  • The hidden state is a fixed-size vector carrying context forward. Same dimensionality every step.
  • The weights are shared across all time steps — the same Wh and Wx are used at every position.
  • The output at step t can be taken at every step (sequence-to-sequence) or only at the end (sequence-to-one).

Weight sharing is what makes RNNs parameter-efficient. A 100-step sentence doesn't need 100× the parameters — the same transformation is applied at every step, learning a universal rule for how new input should update context.

Understanding It Deeply

Three questions that unlock the architecture

The DL Book (Chapter 10) treats the RNN as a dynamical system — a function applied repeatedly to its own output. "Unrolling" the recurrence across time reveals a very deep feedforward network, with one critical difference: the weights are tied across all depths. That tie is both the elegance and the curse.

Think first The hidden state ht has fixed dimensionality — say, 256 numbers — regardless of whether the sequence is 3 steps or 300 steps long. How can a finite vector summarize an arbitrary-length past? What must the network be doing at each step to make this work?

It must be compressing and discarding. Each update to ht is a learned trade-off: what from the old state is worth keeping, what from the new input is worth incorporating, and what to throw away. The network learns this compression from data — whatever is predictive of the target output gets preserved. Whatever isn't gets overwritten. The hidden state is not a memory buffer; it's a learned summary statistic. This is why RNNs can handle variable-length inputs with fixed-size state — but also why long-range dependencies are hard: information from 50 steps ago must survive 50 successive compressions.

Think first In a feedforward network, gradients flow backward through many layers — yet in practice deep feedforward nets (with BatchNorm, residuals) can be trained to hundreds of layers. An RNN unrolled 50 steps is like a 50-layer net with tied weights. Why does the vanishing gradient problem hit RNNs so much harder?

Because the weights are shared. In a deep feedforward net, each layer can learn its own scaling — residual connections let gradients skip directly across layers, and different weight matrices at each depth provide flexibility to avoid collapse. In an RNN unrolled 50 steps, the same Wh matrix is multiplied 50 times in a row during backprop. If the largest eigenvalue of Wh is slightly below 1 — say 0.9 — then after 50 steps the gradient is multiplied by 0.950 ≈ 0.005. It has essentially vanished. No architecture trick at one time step can fix what the shared matrix does over 50 applications. This is the vanishing gradient problem in its purest form — not depth, but repeated multiplication.

Think first Given that the hidden state is a fixed-size learned compression, what kinds of things would you expect a trained RNN to actually store there? What would be worth preserving vs. discarding?

This is one of the more fascinating empirical findings in the DL Book and related literature. Trained RNNs have been shown to track surprisingly interpretable things: whether a quotation is open, whether a sentence is a question, the topic of the current document, the most recent subject for pronoun resolution. But they do this implicitly — the network never decides to "remember the subject." It stores whatever the loss function rewards it for: whatever pattern of past inputs is predictive of the next output. The hidden state dimensions are not human-designed slots for "subject" or "tense" — they are distributed encodings, often polysemantic, always opaque without probing.

The Math

The recurrence equation

Plain English first: the new hidden state is a nonlinear mix of two things — where you were (the old hidden state ht-1) and where you are (the new input xt). The tanh squashes the result into [−1, 1], keeping the state bounded no matter how many steps you run.

RNN Hidden State Update
ht = tanh( Wh · ht-1 + Wx · xt + b )
ht hidden state at time t — the running context vector ht-1 hidden state from the previous step — what the network "remembers" Wh recurrent weight matrix — how much the past state influences the new one Wx input weight matrix — how much the current input contributes xt input at step t — the current token, frame, or observation b bias vector — a learned offset, one per hidden unit tanh hyperbolic tangent — squashes output into (−1, 1), keeps state bounded

Worked example: 3-step sequence, 2D hidden state.

Sentence tokens encoded as scalars: x1 = 0.5, x2 = −0.3, x3 = 0.8. Initial hidden state h0 = [0, 0]. Weight matrices and bias chosen to be small and concrete. All computations rounded to 4 decimal places.

— Wh = [[0.5, 0.1], [0.2, 0.4]]   Wx = [[0.6], [0.3]]   b = [0, 0]
1 h1 = tanh(Wh·[0,0] + Wx·0.5 + 0) = tanh([0.30, 0.15]) → [0.2913, 0.1494]
2 h2 = tanh(Wh·h1 + Wx·(−0.3)) = tanh([0.0761, 0.1181]) → [0.0760, 0.1178]
3 h3 = tanh(Wh·h2 + Wx·0.8) = tanh([0.5196, 0.2709]) → [0.4785, 0.2641]
h3 encodes all three tokens — but the first token's influence is heavily diluted. Each step mixes in new input and compresses the past. After three steps, h1's contribution has been multiplied by Wh twice. At 50 steps that dilution is severe enough to make the earliest tokens nearly unrecoverable — the vanishing gradient problem in miniature.

From Toy to Real

Where RNNs were actually deployed

Before transformers took over in 2017–2018, RNNs were the dominant architecture for any task involving sequences. Three domains defined the era.

Language modeling. Given the words so far, predict the next word. This is the foundational task that most NLP work ultimately reduces to. RNNs (and their LSTM/GRU variants) powered language models, autocomplete, and early machine translation systems. Karpathy's "The Unreasonable Effectiveness of Recurrent Neural Networks" (2015) showed that character-level RNNs could learn structure at the level of XML tags, C code indentation, and even LaTeX formulas — all from raw character sequences, no explicit rules.

Time-series prediction. Stock prices, sensor readings, ECG waveforms, energy demand — any signal sampled over time. The RNN's hidden state accumulates a running estimate of the signal's recent trajectory, making it naturally suited to forecasting. Unlike MLPs that require hand-engineered sliding windows, RNNs learn what horizon to look back over.

Speech recognition. The audio of spoken words is a time series of acoustic features. Deep RNNs stacked with Connectionist Temporal Classification (CTC) loss made it possible to map variable-length audio directly to variable-length text, without manually aligning phonemes to frames. This approach drove commercial speech recognition (Google Voice, Baidu DeepSpeech) before transformers emerged.

Why RNNs were eventually surpassed: They process sequences sequentially — step t must finish before step t+1 can begin. This kills parallelism. Transformers process all positions simultaneously via attention, enabling training on far larger datasets with modern hardware. But RNNs remain relevant for on-device inference where memory is constrained and inputs arrive as true streams.

Common Misconceptions

Five causal misconceptions about RNNs

These follow a chain — each wrong belief leads naturally to the next. The entire chain collapses once you understand weight sharing and what gradient flow actually means.

  • ① The hidden state is like RAM — it stores previous inputs directly.
    The hidden state is a learned compression — not a replay buffer. It doesn't store "The cat sat." It stores whatever numerical summary the training loss rewarded the network for keeping. That might be "a noun phrase with animate subject was the last major constituent" — but expressed as a dense vector, not a discrete record. Information that isn't predictive of future outputs gets overwritten.
  • ② Vanishing gradients are caused by the tanh activation saturating.
    Saturation contributes, but it's not the primary cause in RNNs. The root problem is repeated matrix multiplication of the same Wh. Even with a linear activation (no saturation), if the spectral radius of Wh is less than 1, gradients vanish geometrically with sequence length. This is why LSTMs fix the problem with additive state updates and gating — not by changing the activation function.
  • ③ A larger hidden state means longer memory.
    Hidden state dimensionality controls capacity — how many patterns can be encoded simultaneously. It does not control how far back in time the network can reliably attend. That's governed by gradient flow dynamics, which depend on the weight matrix eigenvalues, not state size. A 1024-dimensional vanilla RNN still suffers vanishing gradients at 50 steps. An LSTM with 128 units handles longer dependencies because of its architecture, not its size.
  • ④ Weight sharing across time steps is a limitation — it would be better to use different weights at each step.
    Weight sharing is the design goal, not a constraint. It allows the network to generalize to sequences of any length — the same rule applies whether a word appears at position 3 or 300. Using different weights at each step (as in a standard feedforward net over a fixed window) would require a fixed maximum length and would not generalize to longer sequences. Weight sharing is what makes RNNs fundamentally different from "just a deep MLP on a window."
  • ⑤ Transformers are just better RNNs — they do the same thing, more efficiently.
    Fundamentally different mechanism. RNNs maintain a fixed-size hidden state that is updated sequentially — information must flow through every intermediate step to reach across long distances. Transformers use attention to directly connect any two positions in the sequence in a single operation — there are no intermediate steps for the information to survive. This makes transformers better at capturing long-range dependencies, but they require seeing the full sequence upfront. RNNs can process true streams where future tokens don't yet exist.

Summary

Five things to lock in

  • The RNN update rule: ht = tanh(Wh·ht-1 + Wx·xt + b). Same weights, every step. The hidden state is a learned compression of all previous inputs.
  • Weight sharing is the feature, not the bug — it lets the same learned rule apply at any position, enabling generalization to sequences of any length.
  • Vanishing gradients arise from repeated matrix multiplication of Wh during backprop. Spectral radius below 1 → geometric gradient decay → early steps receive near-zero gradient signal.
  • The hidden state stores whatever the loss function rewards keeping — not a faithful replay of inputs, but a learned predictive summary. It can be probed but not directly read.
  • RNNs dominated NLP and time-series before 2018 (language modeling, speech recognition, translation). Transformers replaced them for offline tasks, but RNNs remain relevant for true streaming and on-device inference.
One problem this chapter leaves open: You now understand exactly why vanilla RNNs fail at long-range dependencies — the same matrix multiplied over and over, gradients collapsing geometrically. The obvious fix: stop forcing gradient to flow through every intermediate step. Instead, build an architecture with a direct additive path across time, with learned gates controlling what to write, what to erase, and what to output. That's what LSTMs and GRUs do — and it's the next chapter.

Test Your Understanding

9 Questions

0 / 9 revealed

01 An RNN processes a 100-word sentence one word at a time. How many times are the weight matrices Wh and Wx used? ▼
The same Wh and Wx are used at all 100 steps. This is weight sharing — the defining property of RNNs. The same transformation is applied at every position. This is what makes RNNs parameter-efficient and able to generalize to any sequence length, but it's also what causes vanishing gradients: the same matrix is multiplied by itself 100 times during backpropagation.
02 The hidden state ht is always the same fixed size, regardless of how long the sequence is. How can a 256-dimensional vector possibly summarize 500 previous words? ▼
It can't — not faithfully. The hidden state is a lossy learned compression. Each update discards information that the training loss didn't reward preserving. The network learns to keep whatever is predictive of future outputs and overwrite whatever isn't. This is why RNNs struggle with "find the matching parenthesis from 50 steps ago" — that information has likely been compressed away unless the loss explicitly depends on it.
03 The spectral radius of Wh is 0.8. What happens to the gradient signal from step t=50 by the time it reaches step t=1 during backpropagation? ▼
It is multiplied by 0.849 ≈ 0.000024 — essentially zero. The gradient from t=50 must pass through 49 matrix multiplications of Wh to reach t=1. Each multiplication scales the gradient by the spectral radius. With radius 0.8, after 49 steps: 0.849 ≈ 2.4 × 10-5. The parameter at t=1 receives a gradient ~40,000× smaller than the parameter at t=50. This is the vanishing gradient problem — not metaphor, this is the exact arithmetic.
04 Why doesn't exploding gradients also happen, and in fact what condition causes it? ▼
Exploding gradients DO happen — when the spectral radius of Wh exceeds 1. If the largest eigenvalue of Wh is 1.1, then after 50 steps gradients are multiplied by 1.149 ≈ 106. Training becomes numerically unstable. The fix for exploding gradients is simple: gradient clipping (cap the gradient norm at a threshold). Vanishing gradients cannot be fixed this way — you can't amplify gradients to zero. That requires architectural solutions (LSTMs, GRUs, residual connections).
05 An MLP takes a 10-word sentence as a flat 10-word vector. Why can't this approach generalize to 20-word sentences without retraining? ▼
Because the input layer has a fixed size baked into the architecture. An MLP trained on 10-word inputs has a first weight matrix with 10 columns. A 20-word input requires 20 columns — a completely different architecture. RNNs sidestep this by processing one element at a time with the same weights, making the number of steps a runtime parameter, not an architectural constant. This is the core motivation for recurrence on sequential data.
06 Compute one step of the RNN by hand. Given: h0 = [0, 0], x1 = 1.0, Wh = [[0.5, 0], [0, 0.5]], Wx = [[1.0], [0.5]], b = [0, 0]. What is h1? ▼
h1 = [tanh(1.0), tanh(0.5)] ≈ [0.7616, 0.4621]. Pre-activation: Wh·[0,0] = [0,0], Wx·1.0 = [1.0, 0.5], sum + bias = [1.0, 0.5]. Apply tanh: tanh(1.0) ≈ 0.7616, tanh(0.5) ≈ 0.4621. Notice how tanh compresses both values to well within (−1, 1) — if the input were 10.0 instead of 1.0, the pre-activation would be 10.0 but the hidden state would still be near 1.0 due to tanh saturation.
07 A "sequence-to-one" RNN reads an entire sentence and outputs a single sentiment label. A "sequence-to-sequence" RNN outputs a label at every step. Which design would you use for machine translation, and why? ▼
Sequence-to-sequence — but with an encoder-decoder architecture. Machine translation takes a source sentence and produces a target sentence of (potentially) different length. A common approach is to use one RNN to encode the entire source sentence into a fixed context vector (sequence-to-one), then a second RNN to decode that context vector into the target sequence (one-to-sequence). This encoder-decoder pattern was the dominant NLP architecture before attention and transformers replaced the fixed-size bottleneck with a direct attention mechanism over all encoder states.
08 You train a vanilla RNN on a language modeling task. It performs well on sentences up to length 15 but degrades sharply at length 30+. Name the specific cause and what you would try first. ▼
Vanishing gradients preventing learning of long-range dependencies. At length 15, gradient flow is adequate. At length 30+, the early steps receive near-zero gradient and the network cannot learn that events 25 steps ago matter for the current prediction. First thing to try: replace the vanilla RNN with an LSTM or GRU. Both architectures use gating mechanisms that create additive (not multiplicative) paths for gradient to flow directly across long distances, specifically designed to solve this problem.
09 Transformers replaced RNNs for most NLP tasks by 2019. Does this mean RNNs are obsolete? Describe one scenario where an RNN is strictly preferable. ▼
Not obsolete — just no longer the default for offline tasks. RNNs process sequences causally and incrementally: given xt, produce ht and outputt, then move to t+1. Transformers require the full sequence to compute attention over all positions. This makes RNNs strictly preferable for: (1) true real-time streaming where future tokens don't exist yet (live speech, sensor streams, trading signals), (2) on-device inference where the O(n²) attention memory cost of transformers is prohibitive, and (3) any scenario where a fixed-size state is a hard requirement rather than a constraint.
