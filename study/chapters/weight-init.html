Weight Initialization

The decision you make before training even starts — and why it determines whether your network learns anything at all.

← Optimizers · Batch Normalization →
Initialize all weights to zero and your 100-layer network has a 100-layer brain with one thought. Every neuron computes the same thing. Every gradient is identical. Training runs for hours and learns nothing. The symmetry you set at initialization is a cage the network can never escape.

The Intuition

Symmetry breaking and the goal of unit variance

A neural network is a machine for transforming signals layer by layer. Before training begins, you must choose the starting values of every weight. That choice shapes whether signals amplify, collapse, or remain stable as they propagate forward — and whether gradients survive the journey backward.

Why zero = catastrophic failure. If all weights are zero, every neuron in a layer receives the same weighted sum (zero). Every neuron outputs the same activation. During backpropagation, every neuron in the same layer receives the same gradient. Every weight in the same layer receives the same update. After thousands of steps, all neurons in a layer remain perfectly identical — they were symmetric at initialization, and gradient descent, operating on symmetric inputs, can only produce symmetric updates. The symmetry is permanent.

The goal: unit variance propagation. A healthy signal has variance ≈ 1 through the network. If each layer scales the signal up even slightly, a 50-layer network amplifies it by a factor of millions — activations explode. If each layer scales it down, activations vanish to near-zero after just a few layers, and gradients vanish with them. Good initialization keeps variance stable: neither shrinking nor growing as depth increases.

  • Symmetry breaking — weights must differ so neurons specialize into different features.
  • Variance preservation — signal magnitude should not drift as layers increase.
  • Scale matters — too large = exploding activations and gradients; too small = vanishing.
  • Activation-aware — the right initialization depends on which nonlinearity follows.

Understanding It Deeply

Build the intuition from first principles

The Deep Learning Book derives initialization requirements from one key observation: for a network to train, information must be able to flow forward through activations and backward through gradients. Both flows are governed by the same mathematics — matrix multiplication by the weight matrix at each layer.

Think first Why does all-zero initialization cause symmetry that training can never break? Consider: if two neurons in a layer have identical weights, they receive identical inputs and produce identical activations. Backpropagation computes gradients as a function of activations. What does that imply about the updates the two neurons receive?

They receive identical gradients, and therefore identical updates. After the update, they remain identical. This persists forever — gradient descent is deterministic and symmetric inputs produce symmetric outputs. The network effectively has n/layers capacity instead of n. You've built a 1000-neuron network that behaves like a 1-neuron network in each layer. The fix is simple: break symmetry by adding any randomness to weights. Even tiny random values are sufficient.

Think first Why does Xavier initialization assume linear activations while He initialization assumes ReLU? Consider what a ReLU does to variance: it zeros out roughly half the activations (the negative ones). How does this affect the signal's variance compared to a linear activation that passes everything through?

A linear activation passes both positive and negative values through intact — the variance of the output is determined entirely by the weights. But ReLU kills all negative activations, setting them to zero. This effectively cuts the variance in half at every layer. Xavier was derived assuming the full signal passes through (tanh or linear); if you use its formula with ReLU, signal variance is halved at every layer and collapses to zero in deep networks. He initialization corrects for this by using 2/n_in instead of 2/(n_in + n_out) — doubling the variance to compensate for the half that ReLU discards.

Think first What problem do variance-preserving initializations actually solve? Consider what happens when you multiply a random vector by a random matrix 50 times. Does the result stay the same order of magnitude? What does that tell you about what initialization must control?

Each layer multiplication scales the signal by a factor that depends on the weight magnitudes. If that factor is 1.1, fifty layers amplify the signal by 1.1⁵⁰ ≈ 117×. If it's 0.9, fifty layers shrink it to 0.9⁵⁰ ≈ 0.005×. The DL Book formalizes this: the variance of the output of a layer is the variance of the input multiplied by n × Var(w), where n is the fan-in. Setting n × Var(w) = 1 keeps variance stable across arbitrarily many layers. That is exactly what Xavier and He do — they solve for the weight variance that makes this product equal 1.

The Math

Deriving Xavier and He initialization

Both initialization schemes solve the same equation: what weight variance keeps signal variance equal to 1 after one layer? The only difference is the assumed activation function.

Xavier Initialization (Glorot & Bengio, 2010)
Var(w) = 2 / (nin + nout)
nin fan-in — number of incoming connections to this layer nout fan-out — number of outgoing connections from this layer Var(w) the variance to sample each weight from — draw from 𝒩(0, Var(w)) or Uniform(−√3·Var, +√3·Var)
He Initialization (He et al., 2015) — use with ReLU
Var(w) = 2 / nin
nin fan-in — only the input count matters (ReLU kills half the signal, so we compensate) 2/nin the factor of 2 compensates for ReLU zeroing ~50% of activations each layer

A 4-layer network — watch how variance propagates with each scheme:

Assume each layer has n = 512 neurons. Input variance = 1. Activation variance after layer L is computed as (n × Var(w))^L.

LayerZeroLarge σ=1XavierHe (ReLU)
0 1.000 1.000 1.000 1.000
1 0.000 512.0 1.000 1.000
2 0.000 262144 1.000 1.000
3 0.000 1.3×10⁸ 1.000 1.000
4 0.000 6.8×10¹⁰ 1.000 1.000
Pattern: Zero collapses in one step. Large σ=1 explodes exponentially. Xavier and He stay at 1.0 regardless of depth. Xavier achieves this with Var(w) = 2/(n_in + n_out) = 2/(512+512) ≈ 0.00195 — tiny weights, but exactly calibrated so that n × Var(w) = 512 × 0.00195 ≈ 1. He uses Var(w) = 2/512 ≈ 0.0039, doubling it because ReLU halves the effective signal at each layer.

From Theory to Practice

Why pre-trained weights beat any initialization scheme

Everything above assumes you're training from scratch. In practice, the single most impactful "initialization" decision is whether to use pre-trained weights at all — and this choice dominates any difference between Xavier and He.

Transfer learning starts your network not from random noise, but from weights that already represent meaningful features — edges, textures, object parts, syntactic structures. A ResNet-50 pre-trained on ImageNet has weights that encode the visual world. Fine-tuning on your task means those rich representations are your starting point; you're adjusting, not discovering.

The practical trade-off matrix looks like this:

Strategy Data needed Compute Best for
Pre-trained Small OK Low Most real tasks — similar domain, limited data
He / Xavier Large needed High Novel architecture, no pre-trained source available
Zero / const Any Wasted Never — symmetry is unbreakable
The hierarchy in practice: Pre-trained weights > He/Xavier > any other random initialization >> zero/constant. The gap between pre-trained and random is typically 10–20× the gap between He and Xavier. Spend your energy finding the right pre-trained checkpoint — initialization math matters most when that option is unavailable.

Common Misconceptions

Five causal mistakes — and why they chain together

These aren't five random errors. Each one is a logical consequence of the one before it. Trace the chain and none of them can survive.

  • ① Zero initialization is a safe "neutral" starting point — it doesn't bias the network toward anything.
    Zero initialization isn't neutral — it's catastrophic. "Neutral" means not biasing the network toward any particular direction, which random initialization achieves. Zero creates perfect symmetry: every neuron in a layer produces the same output, receives the same gradient, and applies the same update. The network can never escape this symmetry. Neutral would mean small random values that break symmetry without imposing a specific direction.
  • ② So just use large random values — more randomness means more symmetry breaking and better diversity.
    Large random values do break symmetry, but they cause a different failure: exploding activations. If each weight is drawn from 𝒩(0,1) and the layer has 512 inputs, the variance of a neuron's output is 512 × 1 = 512. After two layers: 512² = 262,144. After ten layers: 512¹⁰ ≈ 10²⁷. Activations overflow float32. Gradients follow the same path in reverse — they either explode or vanish. Symmetry breaking requires just enough randomness, not maximum randomness.
  • ③ Then Xavier initialization is always the right choice — it's the principled solution derived from theory.
    Xavier is derived assuming linear activations (or approximately linear, like tanh near zero). It sets n × Var(w) = 1 for the full signal. But ReLU kills roughly half of all activations each layer, effectively halving the variance. Using Xavier with ReLU networks causes the signal to decay by a factor of ~0.5 per layer — a 20-layer network reduces variance by 0.5²⁰ ≈ 10⁻⁶. He initialization corrects for this by using 2/n_in — the factor of 2 compensates for the half that ReLU discards.
  • ④ Good initialization guarantees the network will train well.
    Initialization removes one class of failure (vanishing/exploding signals at step 0), but training failures have many other causes: a learning rate that's too large, optimizer momentum issues, poor data preprocessing, or the loss landscape itself being pathological. Good initialization is a necessary condition for training to proceed — it is not sufficient. And as depth increases, even carefully initialized networks can drift away from unit variance during training — which is why Batch Normalization (next chapter) was invented to enforce variance stability dynamically.
  • ⑤ Initialization doesn't matter much — the optimizer will find the right weights regardless of where you start.
    The optimizer can only follow gradients. If gradients are zero (zero init) or not-a-number (exploding init), there is nothing to follow. The optimizer never gets a chance. Even in less extreme cases, DL Book §8.4 notes that the starting point of optimization matters: it determines which basin of attraction training lands in, and two models initialized differently on the same data can converge to solutions with very different generalization properties. Initialization is not a minor detail that optimization corrects — it shapes the entire trajectory.

Summary

Five things to lock in

  • Zero initialization creates permanent symmetry: all neurons in a layer stay identical forever, regardless of how long you train. The only fix is randomness at initialization.
  • Good initialization preserves signal variance across layers: n × Var(w) = 1. Deviations in either direction cause exponential amplification or collapse with depth.
  • Xavier (2/(n_in + n_out)) assumes linear activations. He (2/n_in) assumes ReLU — the factor of 2 compensates for ReLU zeroing half the activations each layer.
  • The correct scheme depends on the activation function: Xavier for tanh/sigmoid; He for ReLU/Leaky ReLU. Mixing them is a common practical mistake.
  • Pre-trained weights dominate any initialization scheme in practice. When fine-tuning is possible, spend effort choosing the right checkpoint — not choosing between Xavier and He.
One question this chapter leaves unanswered: Good initialization keeps variance stable at step zero. But training changes weights continuously — what ensures variance stays well-behaved after thousands of gradient steps, in a deep network where different layers learn at different rates and the distribution of inputs to each layer shifts throughout training?

That problem is solved by Batch Normalization: it normalizes activations at every layer, on every forward pass, making the network robust to whatever the weights become during training. And that's the next chapter.

Test Your Understanding

10 Questions

0 / 10 revealed

01 You initialize all weights to 0.001 (a very small constant, not zero). Does this solve the symmetry problem? Why or why not? ▼
No — a constant, even small, still creates perfect symmetry. Every neuron in a layer still receives the same weighted sum (0.001 × sum of inputs), produces the same activation, and receives the same gradient. The updates are identical, so after training the neurons remain identical. Symmetry requires that neurons differ, not merely that weights are small. The fix is randomness: draw each weight independently from a distribution so that neurons start in different states.
02 A 10-layer network with n=256 neurons per layer is initialized with weights drawn from 𝒩(0, 1). Estimate the variance of activations at layer 10, assuming linear activations. ▼
256¹⁰ ≈ 1.2 × 10²⁴. Each layer multiplies variance by n × Var(w) = 256 × 1 = 256. After 10 layers: 256¹⁰. This is catastrophically large — float32 can represent at most about 3.4 × 10³⁸, so the network won't necessarily overflow here, but the activations are astronomically large, making loss computation meaningless and gradients useless. Xavier initialization would use Var(w) = 2/(256+256) ≈ 0.0039, keeping n × Var(w) ≈ 1 at every layer.
03 You are building a 20-layer network with ReLU activations. Should you use Xavier or He initialization? What happens if you use the wrong one? ▼
He initialization. Xavier uses Var(w) = 2/(n_in + n_out), designed for linear activations. With ReLU, roughly half the activations are zeroed at each layer, halving the effective variance. With Xavier, each layer multiplies variance by n × (2/(2n)) = 1 for the full signal, but only 0.5 reaches the next layer. After 20 layers, variance is 0.5²⁰ ≈ 10⁻⁶. Activations collapse near zero, gradients vanish, and the network fails to learn. He's formula uses 2/n_in — the factor of 2 exactly compensates for ReLU's half-zeroing.
04 What is the key insight behind the "fan-in / fan-out" formulation in Xavier initialization? Why does the number of connections matter for weight variance? ▼
Each neuron sums over all its inputs, so variance accumulates with fan-in. If a neuron has n_in incoming connections each with variance Var(w), and the inputs are independent with variance 1, the variance of the weighted sum is n_in × Var(w). To keep output variance = 1, we need n_in × Var(w) = 1, giving Var(w) = 1/n_in. Xavier also accounts for the backward pass (where gradients flow through n_out outgoing connections) and averages the two constraints: Var(w) = 2/(n_in + n_out). This keeps both forward activations and backward gradients near unit variance.
05 Two researchers train identical 15-layer ReLU networks from scratch. Researcher A uses He initialization. Researcher B uses Xavier. After 50 epochs, both have the same training loss. Did initialization matter? ▼
Probably — B likely had a much harder time getting there. Xavier with ReLU causes activations to decay through early layers. Early in training, gradients from those layers are near zero, so early layers learn very slowly or not at all. Researcher B likely saw a long period of near-zero progress, and may have required careful learning rate tuning to overcome it. The DL Book notes that good initialization affects not just whether training converges but how quickly — and which solution it finds. Same final loss is not the same training trajectory.
06 Good initialization keeps variance ≈ 1 at step 0. But training changes the weights. What mechanism prevents variance from drifting as training proceeds? ▼
Batch Normalization — the topic of the next chapter. Initialization is a one-time intervention at step 0. As weights update throughout training, the distribution of inputs to each layer shifts (called "internal covariate shift"). Without an ongoing correction mechanism, variance can drift arbitrarily. Batch Normalization addresses this by explicitly normalizing activations to zero mean and unit variance at every layer, on every forward pass, regardless of how the weights have evolved. It makes the network robust to initialization and to variance drift during training.
07 Why is transfer learning typically far more impactful than the choice between Xavier and He initialization? When does the Xavier vs He distinction actually matter? ▼
Pre-trained weights encode learned representations; random weights encode nothing. Xavier and He both solve the variance propagation problem — they give training a clean start. Pre-trained weights go further: they start in a region of parameter space that already represents meaningful features (edges, textures, grammatical structure, etc.). Fine-tuning adjusts these representations; random initialization must discover them from scratch, which requires far more data and compute. Xavier vs He matters when: (1) training from scratch on a novel architecture, (2) no relevant pre-trained checkpoint exists, or (3) you're using activations that specifically require one over the other (ReLU → He; tanh → Xavier).
08 A network with zero-initialized weights produces identical outputs for every input in the training set. Why? And why can't the optimizer fix this? ▼
Because all neurons compute the same function — the network has collapsed to a single neuron per layer. With zero weights, every neuron in layer 1 computes: 0 × x₁ + 0 × x₂ + … = 0, regardless of the input x. The activation of every neuron is the same (e.g. sigmoid(0) = 0.5). Layer 2 also sees identical inputs regardless of what the original input was. The output is the same for every input. The optimizer computes gradients as a function of activations — but since activations are the same for every sample, gradients point in the same direction for every sample, and all neurons receive the same update. The symmetry is self-reinforcing and the optimizer cannot escape it.
09 He initialization uses Var(w) = 2/n_in. Where exactly does the factor of 2 come from? ▼
It compensates for ReLU zeroing approximately half of all activations. Without ReLU, the variance after a layer with n_in inputs and weights of variance Var(w) is n_in × Var(w). Setting this to 1 gives Var(w) = 1/n_in. But ReLU(x) = max(0, x) sets all negative inputs to zero. If the pre-activation values are symmetric around zero (as they roughly are with good initialization), half are positive (pass through) and half are negative (zeroed). The variance of the output is therefore halved: effective variance = 0.5 × n_in × Var(w). Setting this to 1 gives Var(w) = 2/n_in. The factor of 2 undoes the factor of 0.5 that ReLU introduces.
10 A colleague says "I always use He initialization regardless of activation function — it's more conservative and never hurts." Is this good advice? ▼
No — He initialization can cause exploding activations with non-ReLU activations. He uses Var(w) = 2/n_in, which was derived assuming ReLU zeroes half the signal. With a linear activation or tanh, the full signal passes through. The effective variance per layer becomes n_in × (2/n_in) = 2. After L layers, variance is 2ᴸ — after 20 layers, that's over 10⁶. This is the same exploding activation problem that large random initialization causes, just slower. The correct rule is: He for ReLU/Leaky ReLU; Xavier for tanh/sigmoid/linear. Batch Normalization can mitigate this, but it's better to use the right initialization in the first place.
