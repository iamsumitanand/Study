Transformers & Attention

The architecture that made every major AI breakthrough of the last five years possible.

LSTMs read your sentence word by word, holding everything in a single hidden state — a small compressed number that must remember the whole past. Transformers read all words simultaneously, and every word asks every other word: "how relevant are you to me?" No compression. No forgetting. Pure parallel attention.

The Intuition

Attention as a soft lookup table

The transformer's core insight is deceptively simple: to understand a word, look at all other words in the sentence simultaneously and weight each by relevance. This is called self-attention, and it replaces the sequential bottleneck of RNNs with massively parallel computation.

Think of attention as a soft database lookup. You have a query — the word you're currently processing. You compare it against a set of keys — representations of every word in context. The comparison scores become weights, and you use those weights to mix together the corresponding values — the actual content you retrieve. The output is a weighted blend of all values, biased toward the most relevant ones.

  • The Query (Q) — what this token is looking for: "I'm 'bank' — am I a riverbank or a financial bank?"
  • The Key (K) — what each token advertises: "I'm 'river' and I'm highly relevant to river-related queries."
  • The Value (V) — what each token actually contributes when retrieved: the content that gets mixed into the output.

The parallelism advantage is massive. Where an LSTM processes token 1, then 2, then 3 (each step depending on the last), a transformer computes all attention scores for all token pairs at once. On modern GPUs that can do thousands of matrix multiplications in parallel, this makes transformers an order of magnitude faster to train — and it's why they surpassed LSTMs in nearly every benchmark within two years of being introduced.

Understanding It Deeply

Three questions you must be able to answer

The transformer has three design choices that confuse almost everyone the first time. Work through each one — they're not arbitrary, each solves a specific problem.

Think first The input to a transformer is a sequence of token embeddings — each token is just a single vector. Q, K, and V are all computed from that same vector via three separate linear projections. Why would you project the same thing three different ways instead of using the original embedding directly?

Because Q, K, and V serve fundamentally different roles. Using the raw embedding as all three would force a single representation to simultaneously encode "what I'm searching for," "what I offer to others," and "the content I contribute" — these are distinct and often contradictory. The three separate weight matrices allow the model to learn specialized representations for each role. Q can encode intent; K can encode relevance signals; V can encode semantic content. This specialization is not optional — it's what makes attention expressive rather than trivial.

Think first Multi-head attention runs the attention mechanism h times in parallel, each with its own Q, K, V projections, then concatenates the results. A single attention head could in principle learn to attend to anything. What does running multiple heads actually add?

Different heads capture different relationship types simultaneously. In practice, heads specialize: one head might track coreference ("she" → "Alice"), another syntactic dependencies (verb → subject), another positional proximity. A single head would have to compromise across all these patterns. Multi-head attention lets the model maintain multiple attention distributions at once — each head attends to what it finds most useful, and the concatenated output integrates all of them. The "Attention Is All You Need" paper (Vaswani et al., 2017) showed that heads in trained models do systematically specialize in interpretable linguistic relationships.

Think first LSTMs process tokens sequentially, so they automatically encode position — token 1 is processed first, its state is passed to token 2, and so on. But transformers process all tokens in parallel simultaneously. What problem does this create, and what's the fix?

Parallel processing means the transformer has no inherent notion of order. Given tokens "dog bites man" and "man bites dog," attention over the same three tokens would produce identical representations — because without position information, both sequences are just a set of three tokens. The fix is positional encoding: a vector added to each token embedding that encodes its position. The original transformer used sine and cosine functions at different frequencies, so each position gets a unique, smooth pattern. The model can then learn to use positional information just like any other feature. Without it, transformers would be permutation-invariant — blind to word order entirely.

The Math

Scaled dot-product attention

The full attention formula packs four operations into one line: a dot product, a scaling correction, a softmax, and a weighted sum. Each step has an exact reason.

Scaled Dot-Product Attention
Attention(Q, K, V) = softmax( Q·Kᵀ / √dk ) · V
Q Query matrix — one row per token, shape (n × dk) K Key matrix — one row per token, shape (n × dk) V Value matrix — one row per token, shape (n × dv) Q·Kᵀ Raw attention scores — dot product of every query with every key, shape (n × n) √dk Scaling factor — prevents softmax from saturating when dk is large softmax(·) Converts raw scores to a probability distribution over keys (rows sum to 1)

Why divide by √dk? Without scaling, dot products grow large as embedding dimension dk increases — the magnitude of Q·Kᵀ scales with dk. Large values push softmax into regions where gradients are near zero (the function saturates at 0 or 1). Dividing by √dk keeps the variance at ~1 regardless of embedding size, so softmax stays in its useful range. This is not a heuristic — the paper proves the variance of the dot product is dk if Q and K have unit variance, so √dk is the exact corrective factor.

Worked example — 3 tokens, 2D embeddings, compute from scratch:

i Tokens: "cat" (Q=[1,0]), "sat" (K=[0,1]), "mat" (K=[1,1]) — dk=2
1 Compute Q·Kᵀ for query "cat" against all keys:
score("cat","sat") = [1,0]·[0,1] = 0
score("cat","mat") = [1,0]·[1,1] = 1
2 Scale by 1/√dk = 1/√2 ≈ 0.707:
scaled = [0 × 0.707, 1 × 0.707] = [0, 0.707]
3 Apply softmax: e0=1, e0.707≈2.028, sum≈3.028:
weights = [1/3.028, 2.028/3.028] ≈ [0.330, 0.670]
4 Weighted sum of V vectors (Vsat=[0,1], Vmat=[1,1]):
output = 0.330×[0,1] + 0.670×[1,1] = [0.670, 1.000]
"cat" attends 67% to "mat", 33% to "sat". The output blends their values accordingly. With real trained weights, Q, K, V would be learned projections — but the mechanics are identical. Every attention computation reduces to: score → scale → normalize → weighted-mix.

From Architecture to World Impact

One architecture — text, images, audio, code

The transformer was introduced for machine translation. Within five years it had taken over nearly every domain in ML. The same scaled dot-product attention runs inside all of them.

BERT (2018) — Bidirectional Text Understanding Pre-trained by masking random tokens and predicting them. Self-attention is bidirectional — every token attends to all others, past and future. BERT set state-of-the-art on 11 NLP benchmarks simultaneously at launch. The key insight: pre-train on massive unlabeled text, fine-tune on small labeled datasets.
GPT (2018→) — Autoregressive Text Generation Causal (masked) self-attention: each token attends only to previous tokens, never future ones. Trained to predict the next token. Scale this up (GPT-3: 175B parameters, GPT-4: ~1T) and emergent capabilities appear — reasoning, coding, translation — without task-specific training. The same architecture, just bigger and masked differently.
ViT — Vision Transformer (2020) Images split into 16×16 patches, each patch flattened into a vector and treated as a "token." The standard transformer encoder runs on these patch tokens. ViT matched and then surpassed CNNs on ImageNet at scale — proving the architecture isn't just for sequences, it's a general-purpose computation engine.
Audio, Code, Proteins, Video… Whisper (speech-to-text), Codex/GitHub Copilot (code), AlphaFold2 (protein structure), Sora (video generation) — all transformers. The common thread: represent your domain as a sequence of tokens, apply scaled dot-product attention. The architecture generalizes because attention is domain-agnostic.
Why did transformers win so decisively? Three reasons compound: (1) Parallelism enables training on vastly more data than RNNs allow. (2) Attention has no theoretical limit on context length — a token can attend to tokens arbitrarily far away. (3) Scale works — adding parameters and data consistently improves performance with no obvious ceiling. RNNs lacked all three.

Common Misconceptions

Five causal misconceptions — and the chain that connects them

These aren't five unrelated mistakes. Each one leads naturally to the next. Trace the chain and all five become impossible to forget.

  • ① Attention tells you what a word "means."
    Attention tells you which tokens are relevant to a given query — it produces a weighted mixture of values. It does not produce a semantic definition. The output is a context-dependent representation, not a lookup of meaning. "The bank" near "river" and "the bank" near "deposit" produce different attention patterns and different output vectors — both correct uses of the mechanism.
  • ② Then each attention head must learn a single linguistic relationship (like subject-verb).
    Heads do specialize in practice, but there's no mechanism enforcing it. Heads learn whatever mixture of patterns minimizes training loss. Some are interpretable (coreference, syntactic), others aren't. More importantly, heads can change behavior depending on context — the same head doesn't always do the same thing for every input. Claiming heads are cleanly decomposable linguistic detectors is an oversimplification the original authors warned against.
  • ③ Transformers can handle unlimited context because attention spans the whole sequence.
    Attention is O(n²) in sequence length — doubling the context length quadruples compute and memory. In practice, all transformers have a maximum context window (GPT-4: 128K tokens, Gemini 1.5: 1M). Beyond that, tokens simply aren't accessible. Techniques like sliding windows, sparse attention, and retrieval augmentation exist precisely because attention doesn't scale to unlimited length. "Unlimited context" is the single most common misconception about transformers.
  • ④ So the context limit problem is just engineering — given enough compute, transformers remember everything.
    Even within the context window, not all tokens are equally attended to. Empirically, transformers show strong recency bias — recent tokens get disproportionate attention. Information far from the query in the sequence is often effectively lost even when technically in context. This is why "lost in the middle" is a documented failure mode: models with 100K token windows systematically ignore information in the middle of the input.
  • ⑤ Transformers are fundamentally different from everything before them — they're a new paradigm.
    Transformers are built from primitives that existed before 2017: dot-product attention (Bahdanau et al., 2015), residual connections (ResNet, 2015), layer normalization (Ba et al., 2016), and feedforward layers. The insight in "Attention Is All You Need" was eliminating recurrence entirely and showing that attention + position encoding was sufficient. The paradigm shift was architectural, not mathematical — the ingredients were already available.

Summary

Six things to lock in

  • Self-attention computes a weighted sum of all tokens' values, where weights come from query-key dot products. Every token attends to every other in parallel.
  • Q, K, V are three separate linear projections of the same input — each specialized for a different role: what you're searching for, what you offer, what you contribute.
  • Scaling by 1/√dk prevents softmax saturation. Without it, large embedding dimensions would collapse attention into near-one-hot distributions with vanishing gradients.
  • Multi-head attention runs h independent attention operations and concatenates outputs — letting different heads capture different relationship types simultaneously.
  • Positional encoding adds order information to otherwise position-blind token embeddings. Without it, transformers are permutation-invariant — word order is invisible.
  • Transformers have O(n²) complexity in sequence length. Context windows are a hard engineering constraint, not a soft preference — all deployed models have one.
One question this chapter leaves unanswered: Training a transformer from scratch requires enormous amounts of labeled data and compute — neither of which most people have. So how do you take a transformer pre-trained by Google or OpenAI on billions of examples and adapt it to your specific task with only hundreds of examples and a consumer GPU?

That's transfer learning and fine-tuning. And it's the next chapter. Transfer Learning →

Test Your Understanding

9 Questions

0 / 9 revealed

01 Why are Q, K, and V three separate projections of the same input rather than one? What goes wrong if you use the raw embedding as all three? ▼
The three roles are distinct and often contradictory. Q encodes "what I'm searching for," K encodes "what I advertise to others," and V encodes "what content I contribute." Using the raw embedding for all three forces a single representation to serve all roles simultaneously — the model can't specialize. Separate weight matrices let Q, K, V diverge during training to represent each role optimally. Without this separation, attention degenerates into a weaker, less expressive operation.
02 The attention formula divides by √dk. What specific failure mode does this prevent, and why is √dk exactly the right divisor? ▼
It prevents softmax saturation. Without scaling, dot products Q·Kᵀ grow in magnitude with dk — if Q and K have unit variance, their dot product has variance dk, so standard deviation √dk. Large values push softmax into near-zero gradient regions (the function saturates near 0 or 1). Dividing by √dk keeps the variance of the scaled dot product at ~1, maintaining softmax in its useful gradient-flowing range regardless of embedding size. √dk is exact, not approximate.
03 A transformer processes "dog bites man" and "man bites dog" with identical token embeddings but no positional encoding. What do the two sentences produce? ▼
Identical output representations. Without positional encoding, attention computes the same weighted sums regardless of token order — the model is permutation-invariant. Both sentences are just a bag of three tokens with the same embeddings. The outputs would be the same vector for both, making it impossible to distinguish subject from object. Positional encoding (sine/cosine patterns or learned vectors) breaks this symmetry by injecting unique position-dependent information into each token embedding before attention.
04 A model has 8 attention heads, each with dk = 64. You double the number of heads to 16 while keeping total model dimension constant. What happens to dk per head, and what's the trade-off? ▼
dk per head halves to 32. Total model dimension = h × dk = 8 × 64 = 512. With 16 heads: dk = 512/16 = 32. Trade-off: more heads means more diverse attention patterns and relationship types can be captured simultaneously (richer multi-head expressivity), but each head now operates in a lower-dimensional space, potentially reducing its ability to represent complex individual relationships. The original transformer used h=8 and dk=64 as a practical balance — this is a hyperparameter to tune, not a fixed rule.
05 What is causal (masked) self-attention, and why is it necessary for GPT-style language models but not BERT-style models? ▼
Causal attention prevents a token from attending to future tokens by masking them to −∞ before softmax. GPT generates text autoregressively — it predicts token n+1 given tokens 1…n. If token n could attend to token n+1 during training, the model would trivially "look up" the answer rather than learn to predict it. The mask enforces temporal causality. BERT is trained to predict masked tokens from bidirectional context — it never needs to generate sequentially, so all-to-all attention is both valid and beneficial for understanding tasks.
06 A transformer with context length 4K tokens processes a 4K-token document perfectly. You want to process an 8K-token document with the same model without retraining. What breaks, and what are your options? ▼
Positional encodings break. The model has only seen positions 0–4095 during training; positions 4096–8191 are out of distribution. With learned positional embeddings, those positions simply don't exist. With sine/cosine encodings they can extrapolate but quality degrades. Options: (1) truncate to 4K tokens; (2) use sliding window chunking; (3) retrain/fine-tune with longer context; (4) use retrieval-augmented generation to select relevant chunks; (5) upgrade to RoPE (Rotary Position Embedding) or ALiBi which extrapolate better by design.
07 Transformers replaced RNNs/LSTMs in most applications. Name two specific capabilities transformers have that LSTMs structurally cannot have — not just "better," but architecturally impossible for LSTMs. ▼
1. True parallel training across all positions. LSTMs must process token n before token n+1 — each step depends on the previous hidden state. This is a structural sequential dependency that cannot be parallelized regardless of hardware. Transformers compute all positions simultaneously, enabling use of thousands of GPU cores at once. 2. Direct O(1) path length between any two tokens. In an LSTM, information from token 1 must pass through every intermediate hidden state to reach token 100 — a path of length 99, with each step being a potential bottleneck. In attention, token 1 connects directly to token 100 in a single step.
08 ViT (Vision Transformer) applies the standard transformer encoder to images. What must be done to an image to make it compatible with transformer input, and what is the token in this context? ▼
The image is split into fixed-size patches (e.g. 16×16 pixels), each patch flattened into a 1D vector, then linearly projected to the model's embedding dimension. Each patch becomes one "token." A 224×224 image with 16×16 patches produces (224/16)² = 196 tokens plus one [CLS] token for classification — 197 total. Position embeddings are added (patches have 2D spatial positions, so position encoding must capture this). The transformer then runs standard self-attention over these 197 vectors. The key insight: the architecture doesn't care what a token represents — patches are as valid as words.
09 A colleague claims: "Transformers have a 1M-token context window, so they can understand an entire book perfectly." Identify two specific, distinct problems with this claim using concepts from this chapter. ▼
Problem 1 — O(n²) compute cost: A 1M-token context requires attention matrices of size 1M × 1M = 10¹² operations. This is technically feasible only with extreme approximations (sparse attention, chunking). Even when offered, it is vastly more expensive than short contexts — not equivalent compute with more tokens. Problem 2 — "Lost in the middle" degradation: Empirically, transformers show recency bias and attention diffusion over long contexts. Information in the middle of a 1M-token document is systematically underweighted even when technically in context. "Having a 1M context window" ≠ "uniformly attending to everything in that window."
