Logistic Regression

The algorithm that turns a line into a probability — and a probability into a decision.

Linear regression can predict a house price or a temperature. But what happens when you ask it: is this email spam? It will happily output 1.7. Or −0.4. Numbers that mean nothing as a yes/no answer — and nothing as a probability either. The core problem is that linear models produce unbounded outputs, but yes/no questions need outputs pinned between 0 and 1. Logistic regression solves exactly this: it wraps a linear model inside a function that squashes any real number into a probability, then draws a sharp decision boundary. It's the foundational algorithm for binary classification — and the direct conceptual parent of the output layer of every neural network.

The Intuition

The sigmoid: a probability squasher

The sigmoid function takes any real number — from −∞ to +∞ — and maps it to a value strictly between 0 and 1. Feed it a very large positive number and you get something near 1. Feed it a very large negative number and you get something near 0. Feed it zero and you get exactly 0.5. It is, in the most literal sense, a probability squasher.

Here is what makes this useful for classification. Your linear model computes a raw score: z = w₁x₁ + w₂x₂ + … + b. That score can be anything. But once you pass it through the sigmoid — written σ(z) — the output becomes a well-behaved probability. σ(z) > 0.5 means the model leans toward class 1. σ(z) < 0.5 means it leans toward class 0. The decision boundary is exactly where z = 0.

The sigmoid is smooth everywhere. Its gradient is never zero away from the extremes. This is critical: it means gradient descent can always find a direction to improve. A hard threshold (output 0 below 0.5, output 1 above) has zero gradient everywhere — gradient descent would be useless. The sigmoid is the continuous, differentiable approximation that makes learning possible.

  • The linear score z can be any real number — positive means "looks like class 1," negative means "looks like class 0."
  • The sigmoid σ(z) converts z into a probability between 0 and 1.
  • The decision boundary is the line (or hyperplane) where σ(z) = 0.5, i.e., where z = 0.
  • The loss function (cross-entropy) penalizes confident wrong predictions enormously — far more than linear loss would.

Understanding It Deeply

Three questions most explanations skip

Logistic regression is often presented as a recipe: take linear model, apply sigmoid, minimize cross-entropy. But the real understanding comes from asking why each ingredient is necessary — and what goes wrong without it.

Think first Why can't you just apply a threshold to linear regression? Output 1 if the prediction exceeds 0.5, output 0 otherwise? It's simpler. What specifically goes wrong?

Two things go wrong. First, the output is no longer differentiable — it jumps from 0 to 1 with a gradient of zero everywhere except the threshold, making gradient descent blind. The model would have no signal about how to improve. Second, even if you somehow trained it, the probabilities are meaningless. Linear regression can output 1.7 for a point that should be class 1, and −0.2 for one that should be class 0. After thresholding these are both technically "correct" predictions, but the model has no incentive to push the outputs toward actual probabilities. You lose calibration: the number 0.7 should mean "70% chance of class 1." A thresholded linear model gives you no such guarantee.

Think first Cross-entropy loss penalizes a prediction of 0.01 for a true-label-1 example far more severely than a prediction of 0.49. Mean squared error would give them similar penalties. Why does this asymmetry matter?

Because wrong and confident is catastrophically worse than wrong and uncertain. If your spam filter outputs 0.49 for a spam email, it's borderline — understandable. If it outputs 0.01, it is actively asserting "I am 99% certain this is not spam." That is a serious failure of a different kind. Cross-entropy's log penalty grows to infinity as the predicted probability approaches 0 for a true-positive case. The model is punished without limit for confident mistakes. This is exactly the right behavior for probabilistic classifiers: hedging uncertainty is always safe; false certainty is not.

Mathematically: for a true label y = 1, the loss is −log(ŷ). If ŷ = 0.99, loss ≈ 0.01. If ŷ = 0.5, loss ≈ 0.69. If ŷ = 0.01, loss ≈ 4.6. The penalty grows exponentially as the model becomes confidently wrong. MSE would give losses of 0.0001, 0.25, and 0.9801 respectively — a much gentler and informationally weaker signal.

Think first "Logistic regression" contains the word "regression" yet it is used for classification. This confuses almost everyone. What is actually being "regressed" here?

The probability. Logistic regression is, literally, regression on a probability. The model regresses the linear score z to the probability P(y=1 | x) via the sigmoid function. The name predates the modern use of "classification" as a distinct task category — the technique was developed in statistics in the 1950s where "regression" meant fitting a curve to data. The logistic in the name refers to the logistic function, which is an older name for the sigmoid. So the name is historically accurate: it is a regression that uses the logistic function. The output just happens to be a probability, which makes it ideal for classification.

The Math

Three formulas — and what they mean

Every logistic regression derivation uses the same three building blocks: the sigmoid, the log-likelihood, and the gradient update. Understand each one in isolation first, then watch them lock together.

The Sigmoid Function
σ(z) = 1 / (1 + e^{−z})
z the linear score: z = w·x + b — can be any real number σ(z) the predicted probability P(y=1 | x) — always in (0, 1) e^{−z} grows large when z is very negative, driving σ(z) → 0
Log-Likelihood (Cross-Entropy Loss)
L = −(1/n) · Σ [ y · log(σ(z)) + (1−y) · log(1−σ(z)) ]
y true label for this example: 0 or 1 σ(z) model's predicted probability y=1 only the first term survives: penalize −log(σ(z)), grows ∞ as σ(z)→0 y=0 only the second term survives: penalize −log(1−σ(z)), grows ∞ as σ(z)→1
Gradient Update (one weight w_j)
w_j := w_j − α · (1/n) · Σ (σ(z_i) − y_i) · x_{ij}
σ(z_i) − y_i prediction error on example i — exactly as in linear regression x_{ij} the j-th feature of example i — how much this feature contributed to the error α learning rate — how far to step each update

The gradient update form is remarkable: it is identical in structure to linear regression's update rule. The sigmoid disappears in the gradient — it was absorbed during differentiation of the log-likelihood. This is not a coincidence: the cross-entropy loss was chosen precisely because its gradient with respect to the sigmoid has this clean form. The sigmoid and the log-loss are conjugate pairs.

Four steps — spam/not-spam with real numbers:

Setup: classify emails as spam (y=1) or not (y=0). Features: x₁ = word count / 100, x₂ = exclamation count. Initial w = [0.5, 0.3], b = −0.2, α = 0.5. Example email: 3 exclamations, 250 words → x = [2.5, 3.0], true label y = 1 (spam).
1 Compute linear score z   z = 0.5 × 2.5 + 0.3 × 3.0 + (−0.2) = 1.25 + 0.90 − 0.20 = 1.95
2 Apply sigmoid   σ(1.95) = 1 / (1 + e^{−1.95}) ≈ 1 / (1 + 0.142) ≈ 0.875  → model says 87.5% chance spam
3 Compute loss   L = −log(0.875) ≈ 0.134  → small, prediction was confident and correct
4 Update weights   error = σ(z) − y = 0.875 − 1.0 = −0.125
Δw₁ = −0.5 × (−0.125) × 2.5 = +0.156  →  w₁ = 0.656
Δw₂ = −0.5 × (−0.125) × 3.0 = +0.188  →  w₂ = 0.488
Both weights increased — they will push z even higher for high-exclamation, long emails. The update direction is identical to linear regression: error × feature. The sigmoid already did its job in step 2. The gradient of cross-entropy through sigmoid simplifies to (σ(z) − y) · x — the sigmoid cancels out in the derivative. This clean form is the mathematical reason logistic regression is trainable.

From Toy to Real

Email spam detection: from 2 features to 10,000

The toy example above used two features. Real spam detection uses 10,000 or more — one per word in the vocabulary (a "bag of words" model). Each feature is the count of how many times that word appears in the email.

The structure of logistic regression doesn't change. The linear score is still z = w·x + b — now w is a 10,000-dimensional vector. The sigmoid still squashes z to a probability. Cross-entropy is still the loss. Gradient descent still runs the same update rule.

What does change: the decision boundary. In 2D it was a line. In 10,000D it is a hyperplane — a flat 9,999-dimensional surface cutting through feature space. Every point on one side is classified as spam; every point on the other side is not-spam. Logistic regression is still a linear classifier — its decision boundary is always a flat surface, regardless of dimensionality.

Why linear classification still works at 10,000 dimensions At first glance it seems too simple — a hyperplane deciding spam vs. not-spam? But consider: in 10,000 dimensions, the word "viagra" has its own axis, "congratulations" has its own axis, "invoice" has its own axis. The hyperplane can weight each word independently. The combination of thousands of independently-weighted signals creates a surprisingly powerful separator. Logistic regression with bag-of-words features was the state of the art for spam detection well into the 2010s, and still outperforms neural networks on small datasets due to its simplicity and resistance to overfitting.

The key limitation: logistic regression cannot learn interactions between features. It cannot discover "free + winner" is spammier than either word alone (unless you manually add a "free_winner" feature). That's what neural networks learn to do — and exactly why the next chapter introduces decision trees, which can split on feature interactions explicitly.

Common Misconceptions

Five wrong beliefs — and the chain that connects them

These aren't five random mistakes. Each one flows from the previous. Follow the chain and they all become impossible to forget.

  • ① Logistic regression outputs the predicted class label.
    It outputs a probability — a number between 0 and 1. The class label is obtained by applying a threshold (default: 0.5), but the model itself never produces a hard 0 or 1. This distinction matters enormously: you can tune the threshold based on the cost of false positives vs. false negatives, which a hard-label model makes impossible.
  • ② So the 0.5 threshold is always the right one to use.
    Only if false positives and false negatives have equal cost. In medical diagnosis, missing a cancer (false negative) is far more costly than a false alarm (false positive) — use a lower threshold like 0.2. In spam detection, filtering a legitimate email (false positive) may be more annoying than letting spam through — use 0.7. The optimal threshold is a business decision, not a mathematical one.
  • ③ Then surely logistic regression can't handle nonlinear class boundaries.
    The model's decision boundary in the original feature space is always linear. But you can engineer nonlinear features — x², x·y, log(x), sin(x) — and pass them in as inputs. Logistic regression will draw a linear boundary in this transformed space, which corresponds to a curved boundary in the original space. Neural networks learn these feature transformations automatically; logistic regression requires you to choose them manually.
  • ④ So neural networks have made logistic regression obsolete.
    Logistic regression is the output layer of every neural network trained for binary classification. The final layer applies a sigmoid to the learned representation and minimizes cross-entropy — that is literally logistic regression applied to neural features. Understanding logistic regression is understanding what the network is doing in its final step. More practically: on tabular data, logistic regression is fast, interpretable, and often competitive with neural networks.
  • ⑤ At least cross-entropy and log-loss are different things worth distinguishing.
    They are the same thing for binary classification, just named from different traditions. "Log-loss" comes from Kaggle/ML engineering culture and refers to the average cross-entropy per example. "Cross-entropy loss" comes from information theory. "Negative log-likelihood" comes from statistics. All three compute −[y·log(ŷ) + (1−y)·log(1−ŷ)]. The conceptual chain (① → ⑤) ends here: hard labels → thresholds → nonlinearity → neural nets → loss functions. They form one coherent story.

Summary

Five things to lock in

  • The sigmoid σ(z) = 1/(1+e^{−z}) squashes any real number to (0,1). It is the bridge between linear scores and probabilities. The decision boundary is always where z = 0.
  • Cross-entropy loss = −[y·log(ŷ) + (1−y)·log(1−ŷ)]. It penalizes confident wrong predictions with unbounded severity. Mean squared error would give far weaker learning signal for classification.
  • The gradient update reduces to (σ(z) − y)·x — identical in form to linear regression. The sigmoid and log-loss are mathematically conjugate: the sigmoid cancels out when you differentiate the loss.
  • Logistic regression is a linear classifier. Its decision boundary is always a hyperplane. It cannot learn feature interactions without manual feature engineering — that's what neural networks and decision trees do.
  • The output layer of every binary classification neural network is logistic regression. The model learns a good feature representation; logistic regression draws the final boundary. It is not obsolete — it is foundational.
One limitation this chapter reveals: Logistic regression draws a flat decision boundary — always a hyperplane. Real-world data is often not linearly separable. You need a model that can ask "is x₁ large AND x₂ small?" — a conditional split, not a weighted sum. A model that can recursively partition the feature space into regions, each making its own local prediction. That's a decision tree. And a forest of decision trees — each trained on a random subset of the data and features — is one of the most powerful practical ML algorithms in existence for tabular data.

Test Your Understanding

10 Questions

0 / 10 revealed

01 What does σ(0) equal, and what does that tell you about the decision boundary? ▼
σ(0) = 0.5. Plugging z = 0 into 1/(1+e^{−z}) gives 1/(1+1) = 0.5. The decision boundary is exactly the set of points where the linear score z = w·x + b equals zero. On one side, z > 0 and the model outputs probability > 0.5 (predicts class 1). On the other side, z < 0 and the model predicts class 0. The decision boundary is the hyperplane z = 0 — a line in 2D, a plane in 3D, a hyperplane in higher dimensions.
02 Your model predicts 0.02 for a true-label-1 example. Calculate the cross-entropy loss and explain why it is so high. ▼
Loss = −log(0.02) ≈ 3.91. The formula for y=1 is −log(ŷ). log(0.02) ≈ −3.91, so the loss is 3.91. This is extremely high because the model is confidently wrong — it assigned only 2% probability to the correct class. Cross-entropy's log penalty grows without bound as the prediction approaches 0 for a true positive, precisely to punish confident mistakes. Compare: if the model had predicted 0.5, loss = −log(0.5) ≈ 0.69. Confident errors cost orders of magnitude more than uncertain ones.
03 The gradient update for logistic regression looks like (σ(z_i) − y_i) · x_ij. Why is the sigmoid not present in this expression, even though the model uses a sigmoid? ▼
The sigmoid cancels algebraically when you differentiate cross-entropy. When you take ∂L/∂w_j through the chain rule — differentiating the log-likelihood through the sigmoid — the derivative of the sigmoid (σ(z)(1−σ(z))) appears in the numerator, and the same term appears from the log in the denominator. They cancel exactly, leaving just (σ(z) − y) · x. This clean cancellation is why the sigmoid and cross-entropy are called a "conjugate pair" — they were designed to work together. Using sigmoid with MSE, for example, does not produce this clean gradient.
04 You're building a cancer detection classifier. The dataset has 95% negative samples and 5% positive (cancer) cases. Why is accuracy a bad metric, and what would you use instead? ▼
A model that predicts "no cancer" for every patient achieves 95% accuracy while completely failing at its job. With a 95/5 split, accuracy rewards the trivial "always negative" strategy. Better metrics: AUC-ROC (measures discrimination across all thresholds), precision-recall AUC (better for rare positives), or direct reporting of sensitivity (true positive rate) and specificity (true negative rate). For this application you'd also deliberately set the classification threshold well below 0.5 — missing a cancer is far more costly than a false alarm, so you'd accept many false positives to avoid false negatives.
05 A friend says "logistic regression is just linear regression with a sigmoid on top." Is this accurate? What does it miss? ▼
Partially accurate, but misses the loss function — which is the crucial difference. The model architecture is: linear score → sigmoid → probability. That part is correct. But if you keep linear regression's MSE loss, you get a non-convex loss surface with many bad local minima, and poor probability calibration. Logistic regression replaces MSE with cross-entropy (negative log-likelihood), which: (1) makes the optimization convex — gradient descent is guaranteed to find the global minimum, (2) produces proper probability calibration, and (3) gives the clean gradient that makes training stable. The sigmoid + cross-entropy combination is what makes logistic regression work as a classifier, not just as a modified regressor.
06 You train logistic regression on a 2D dataset where the classes form two concentric circles (inner circle = class 0, outer ring = class 1). The model fails completely. Why, and what's the simplest fix? ▼
Because a linear decision boundary (line in 2D) cannot separate concentric circles — the boundary would need to be circular. Logistic regression is a linear classifier: its decision boundary is always z = w₁x₁ + w₂x₂ + b = 0, which is a straight line. Any linear boundary will split one circle in half, not separate the inner from the outer. Simplest fix: add the feature r² = x₁² + x₂². In this augmented 3D feature space (x₁, x₂, r²), the classes become linearly separable — the boundary is a plane at r² = constant, which corresponds to a circle in the original space. This is feature engineering for nonlinear classification.
07 What is the connection between maximum likelihood estimation and minimizing cross-entropy loss in logistic regression? ▼
They are exactly equivalent — minimizing cross-entropy is maximizing the log-likelihood of the training data. Under the model's probability P(y=1|x) = σ(w·x+b), the likelihood of observing the training labels is the product of P(y_i|x_i) across all examples. Taking the log converts the product to a sum: log P(y_i=1|x_i) = log σ(z_i) when y_i=1, and log(1−σ(z_i)) when y_i=0. Negating this sum (to turn maximization into minimization) gives exactly the cross-entropy loss formula. Maximum likelihood is the statistical story; cross-entropy is the optimization story. They're the same thing.
08 What happens to the sigmoid curve and the decision boundary when you increase the magnitude of the weight vector w (while keeping its direction the same)? ▼
The decision boundary stays in the same place, but the sigmoid transition becomes sharper — the model becomes more confident everywhere. The decision boundary is where w·x + b = 0. Scaling w by a constant k gives kw·x + kb = 0, which is the same set of points. But the sigmoid σ(kz) has steeper slope at the boundary — it transitions from near-0 to near-1 over a smaller region. At very large k, σ(kz) approaches a step function: essentially 0 on one side, 1 on the other. This is what happens when logistic regression is trained on linearly separable data without regularization — the weights diverge to infinity as the model tries to make its predictions more and more certain.
09 How is softmax regression (multiclass logistic regression) related to binary logistic regression? What specifically generalizes? ▼
Softmax generalizes the sigmoid from 2 classes to K classes by computing a normalized probability distribution over all K outcomes. In binary logistic regression: σ(z) = e^z / (e^z + e^0) — probability of class 1 vs. class 0. Softmax extends this: P(class k) = e^{z_k} / Σ_j e^{z_j}. With K=2, softmax reduces to the sigmoid (the binary case is a special case). The loss is still cross-entropy: −log(ŷ_{true class}). The gradient is still (predicted probability − true label) · x for each class. As CS231n notes: "The Softmax classifier provides calibrated probability estimates for each class" — same probabilistic interpretation, generalized to multiple classes.
10 You train logistic regression and get 92% training accuracy and 91% test accuracy. A neural network gets 99% training and 91% test accuracy. Which model would you prefer in production, and why? ▼
Logistic regression — same test performance, dramatically simpler. Both models achieve the same test accuracy (91%), which is what matters in production. The neural network has overfit to the training data (99% vs. 91% gap). Logistic regression has matched its training accuracy on test data — excellent generalization. For production, logistic regression wins on: interpretability (you can examine each weight and see what drives predictions), inference speed (one dot product + sigmoid vs. many layers), debuggability, and robustness to distribution shift. The neural network's extra training accuracy bought nothing useful. Always prefer the simpler model when test performance is equal.
