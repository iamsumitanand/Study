Part 2 · Neural Networks

Batch Normalization

The technique that made training 100-layer networks practical.

Internal Covariate Shift: the problem that made 100-layer networks impossible to train before 2015.

Before Batch Normalization, deep networks were notoriously unstable. Each layer's inputs shifted in distribution every time the layer below it updated its weights. Training had to crawl with tiny learning rates. Careful weight initialization was fragile. Networks with more than about 20 layers simply wouldn't converge reliably. Then Ioffe and Szegedy published one paper — and training deep networks became almost embarrassingly easy.

The Intuition

Why a stable input distribution speeds up learning

Imagine training a neural network where each layer must simultaneously learn its own transformation and adapt to the moving target of its inputs — because the layer below is also updating. Batch Normalization breaks this coupling. It standardizes each layer's inputs, giving every layer a stable surface to learn on, regardless of what happened below.

The core observation is simple: if you normalize the inputs to a neural network, training speeds up. That principle was already known. Batch Normalization extends it to the internal activations of a deep network — to every layer, not just the first.

For a mini-batch of m examples flowing through a particular layer, BN computes the mean and variance of the pre-activation values across the batch, then normalizes them to have zero mean and unit variance. The result: the inputs to each layer's activation function are always centered and scaled to a predictable range — no matter how the weights in earlier layers are evolving.

  • The covariate shift problem — the distribution of inputs to each layer changes throughout training as earlier-layer parameters change. BN pins this distribution.
  • A stable distribution means larger learning rates are safe, reducing training time dramatically.
  • Initialization sensitivity drops because BN corrects for poorly-scaled initial weights automatically.
  • BN introduces a subtle regularization effect — each sample's normalization depends on other samples in the batch, adding noise that acts like dropout.

Understanding It Deeply

Three questions worth thinking through

Think first After normalizing to zero mean and unit variance, every neuron's output has the same scale. Why might this actually hurt the network? What representational power does normalization destroy — and how does BN recover it?

Normalization removes the mean and variance that the layer may have learned intentionally. If a sigmoid layer learned to saturate (stay near 1 for most inputs), normalizing destroys that. To recover this power, BN adds two learnable parameters per feature: γ (scale) and β (shift). After normalization, it applies y = γ·x̂ + β. If the network needs the original distribution back, it can learn γ = σ and β = μ and undo the normalization entirely. The key insight: BN constrains the initial form of the distribution (zero mean, unit variance) while still allowing the network to learn any distribution it needs. It's a parameterized normalization.

Think first During training, BN normalizes using the mean and variance of the current mini-batch. Why can't it do the same thing at inference time? What changes, and what does BN use instead?

At inference, you might be predicting on a single example — there's no batch to compute statistics over. Even if you did batch inference, the statistics would change for every batch, making outputs non-deterministic. So during training, BN also accumulates running estimates of the population mean and variance using a moving average. At inference, these fixed statistics are used instead of per-batch statistics. This means BN behaves differently in training mode vs. inference mode — a crucial distinction that every ML framework (PyTorch, TensorFlow) implements as model.train() vs. model.eval(). Forgetting to switch modes is a classic bug that produces suspiciously good training metrics and terrible real-world performance.

Think first BN is often described as a regularizer — yet it was designed to stabilize training, not prevent overfitting. What property of BN produces a regularization effect as a byproduct?

When BN normalizes a sample, it uses the mean and variance computed from the entire mini-batch, not just that sample. This means each example's normalized value depends on which other examples happen to be in the same batch. Every time the network sees the same training example, it gets a slightly different normalized representation — because the batch composition changes. This stochasticity acts like a noise injection, similar in spirit to dropout. The effect is strong enough that when using BN, dropout is often unnecessary. Ioffe and Szegedy's original paper notes this as a secondary benefit but it becomes primary in practice: BN lets you reduce or eliminate other regularization.

The Math

BN forward pass — four steps

Batch Normalization is applied to a feature across a mini-batch of m examples. For each feature dimension independently: compute the batch mean, compute the batch variance, normalize, then apply the learnable affine transform.

Batch Normalization Forward Pass
μ_B = (1/m) · Σᵢ xᵢ
σ²_B = (1/m) · Σᵢ (xᵢ − μ_B)²
x̂ᵢ = (xᵢ − μ_B) / √(σ²_B + ε)
yᵢ = γ · x̂ᵢ + β
m mini-batch size — number of examples μ_B batch mean — average pre-activation value across the batch σ²_B batch variance — spread of the pre-activation values ε small constant (e.g. 1e-5) for numerical stability — prevents division by zero x̂ᵢ normalized value — zero mean, unit variance (before affine transform) γ, β learnable scale and shift — recovered representational power, trained by backprop yᵢ BN output — normalized then rescaled; this feeds into the activation function

Worked example: 4-sample mini-batch through one BN layer.

Pre-activation inputs: x = [2, 4, 6, 8]. Learnable parameters initialized to γ = 1, β = 0.

1 Compute batch mean: μ_B = (2 + 4 + 6 + 8) / 4 = 20 / 4 = 5.0
2 Compute batch variance: σ²_B = [(2−5)² + (4−5)² + (6−5)² + (8−5)²] / 4
= [9 + 1 + 1 + 9] / 4 = 20 / 4 = 5.0
3 Normalize each input (ε ≈ 0): x̂ = (x − 5.0) / √5.0 = (x − 5.0) / 2.236
x̂ = [−1.342, −0.447, +0.447, +1.342] mean(x̂) = 0.0 · std(x̂) = 1.0 ✓
4 Apply affine transform (γ=1, β=0 at init): y = 1 · x̂ + 0 = x̂
With γ=2, β=1: y = 2·x̂ + 1 = [−1.684, +0.106, +1.894, +3.684] The affine transform restores representational power. A network that needs outputs near [2,4,6,8] can learn γ ≈ 2.236 and β ≈ 5 — recovering the original distribution exactly. BN constrains only the initial form; it doesn't lock the network into zero mean forever.

In Practice

ResNets, Transformers, and when BN breaks down

Batch Normalization powered the deep learning revolution in computer vision — enabling architectures like ResNet (152 layers) that were simply not trainable before. But it has known failure modes, and modern architectures often replace it.

ResNets (He et al., 2015). Batch Normalization is placed after every convolutional layer and before the activation function. The combination of BN + skip connections made 50-, 101-, and 152-layer networks train stably and converge faster than shallower VGG-style networks. BN is why modern CNNs don't need careful hand-tuning of learning rates and initializations — it provides a kind of automatic gradient highway.

Transformers use Layer Normalization, not Batch Normalization. The critical difference: LayerNorm normalizes across the feature dimension for each individual example independently — not across the batch. This means LayerNorm works correctly with a batch size of 1. For language modeling this matters enormously: sequences have variable lengths, and inference is often done on single sentences. BatchNorm's statistics become noisy (and meaningless) at very small batch sizes. LayerNorm's statistics are computed entirely within a single example, so they are always well-defined regardless of batch size.

BatchNorm vs LayerNorm — the key distinction:

BatchNorm normalizes across the batch dimension (all examples, one feature at a time). It breaks with batch size = 1 and behaves differently between training and inference.

LayerNorm normalizes across the feature dimension (all features of one example at a time). It works identically at any batch size, and training/inference behavior is the same. This is why every modern transformer — GPT, BERT, LLaMA — uses LayerNorm.

When BN hurts: Recurrent networks (RNNs, LSTMs) suffer from BN because sequence lengths vary — the batch statistics at each time step depend on which sequences happen to be the same length in the batch. GroupNorm and InstanceNorm were developed specifically to handle cases where batch statistics are unreliable.

Common Misconceptions

Five wrong beliefs about Batch Normalization

BN is one of the most misunderstood techniques in deep learning. These five misconceptions are particularly persistent — and all trace back to not reading the original paper carefully.

  • ① BN forces every layer's output to have mean=0 and std=1.
    Only after the normalization step. The affine transform y = γ·x̂ + β immediately undoes this. The network can learn any mean and variance it needs via γ and β. Saying "BN normalizes activations to zero mean" is only half the story — the second half is that it then immediately rescales them with learnable parameters.
  • ② BN works the same way at training time and inference time.
    It uses fundamentally different statistics. During training, BN uses the mean and variance of the current mini-batch. During inference, it uses running estimates (moving averages) of the population mean and variance accumulated during training. Forgetting to call model.eval() in PyTorch — or model(training=False) in Keras — is a classic silent bug: the model computes batch statistics on whatever test data comes in, which are wrong, and the outputs become subtly incorrect.
  • ③ BN eliminates the need for careful weight initialization.
    BN greatly reduces initialization sensitivity — but does not eliminate it. Very poor initialization (e.g., all-zero weights, or extremely large variance) can still cause issues, particularly in the first forward pass before BN has any gradient signal to work with. BN also does not help with symmetry breaking — if all neurons in a layer are initialized identically, BN normalizes them to the same values, perpetuating the symmetry problem.
  • ④ Batch size doesn't matter for BN — it just normalizes whatever it gets.
    Batch size is critical. With batch size = 1, BN computes mean and variance from a single example, which is meaningless. Even small batches (2–4) produce extremely noisy estimates. This is the primary reason transformers use LayerNorm: language model training often involves batch sizes of 1 for inference, and very small effective batch sizes per sequence at training time. GroupNorm and InstanceNorm were designed specifically to work at small batch sizes.
  • ⑤ BN was introduced to fix the vanishing gradient problem.
    BN was explicitly designed to fix internal covariate shift — the instability of each layer's input distribution caused by changing weights in earlier layers. Vanishing gradients were a separate, earlier problem addressed by better activation functions (ReLU), skip connections, and initialization schemes like Xavier/He. BN does help gradients flow more cleanly as a byproduct, but conflating the two problems leads to confusion about when each fix applies.

Summary

Five things to lock in

  • BN normalizes pre-activations to zero mean and unit variance per feature per mini-batch, then applies a learnable affine transform γ·x̂ + β to recover representational power.
  • γ and β are not fixed — they are trained by backpropagation. The network can learn to completely undo the normalization if needed; BN just constrains the starting point.
  • Training mode uses per-batch statistics; inference mode uses running averages accumulated during training. Forgetting to switch modes is a classic silent bug.
  • BN degrades at small batch sizes because per-batch statistics become noisy. LayerNorm (used in transformers) normalizes across the feature dimension per example — no batch dependency.
  • BN's regularization effect comes from the stochasticity of batch statistics — each example is normalized relative to different batch-mates each epoch, injecting noise that reduces overfitting.

9 Questions

0 / 9 revealed

01 After BN normalizes a layer's inputs to zero mean and unit variance, it immediately applies y = γ·x̂ + β. What is the point of this affine transform if we just carefully normalized the inputs? ▼
To restore representational power. Normalization removes the mean and variance that the network may have learned intentionally. The affine transform lets the network learn any mean and variance it needs — if the optimal distribution happens to be the original one, it can learn γ = σ_original and β = μ_original to undo the normalization. BN doesn't constrain what distribution the network uses; it only constrains the form before the affine transform. γ and β are trained by backpropagation just like weights.
02 You train a model with BN and it achieves 95% validation accuracy. You deploy it, but predictions are subtly wrong. Training and deployment code look identical. What is the most likely cause? ▼
The model is running in training mode instead of inference mode. In training mode, BN computes statistics from the current input batch. At inference on production data, those batch statistics are computed over whatever requests happen to arrive together — which differ from the training distribution. The fix: call model.eval() in PyTorch (or equivalent) before inference. This switches BN to use the running mean and variance accumulated during training, which are stable population estimates.
03 Mini-batch inputs to a BN layer: x = [1, 3, 5, 7]. Compute μ_B, σ²_B, and the normalized values x̂. ▼
μ_B = 4.0 · σ²_B = 5.0 · x̂ = [−1.342, −0.447, +0.447, +1.342].
μ_B = (1+3+5+7)/4 = 16/4 = 4.0.
σ²_B = [(1−4)² + (3−4)² + (5−4)² + (7−4)²] / 4 = [9+1+1+9]/4 = 20/4 = 5.0.
√σ²_B = √5 ≈ 2.236.
x̂ = [(1−4)/2.236, (3−4)/2.236, (5−4)/2.236, (7−4)/2.236] = [−1.342, −0.447, +0.447, +1.342].
04 Transformers use LayerNorm, not BatchNorm. What is the single most important reason for this choice? ▼
BatchNorm fails at batch size = 1, which is standard at inference time for language models. BatchNorm normalizes across the batch dimension — with one example, you compute mean and variance of a single number, which is undefined or meaningless. LayerNorm normalizes across the feature dimension within a single example, so it works identically regardless of batch size. Since language model inference is typically done one token sequence at a time, and sequence lengths vary, LayerNorm is the only practical choice.
05 BN is described as a regularizer even though it was designed for training stability. What property of BN produces a regularization effect? ▼
Each example is normalized relative to its batch-mates, which changes every epoch. The normalized value of a training example depends on the mean and variance of the other samples in the same mini-batch. Since batch composition changes each epoch, the same example receives slightly different normalized representations each time. This acts as a noise injection — similar to dropout — that prevents the network from memorizing exact patterns and improves generalization. Ioffe and Szegedy note this allows reducing or eliminating dropout in many architectures.
06 BN is typically placed between a linear transformation and an activation function (e.g., Conv → BN → ReLU). Why this order and not Conv → ReLU → BN? ▼
Normalizing before the activation makes the normalization act on the full, symmetric pre-activation distribution. Before ReLU, the values can be positive or negative, so mean and variance are well-defined and meaningful to normalize. After ReLU, all negative values are zeroed — the distribution is truncated and asymmetric. Normalizing a truncated distribution is less meaningful and can actually hurt because the clipped negatives distort the estimated variance. The original paper places BN on the pre-activation values explicitly for this reason.
07 You initialize γ = 0 and β = 0 for all BN layers. What happens on the first forward pass, and why is this a problem? ▼
Every layer outputs exactly zero — the network cannot learn anything. With γ = 0, the BN output y = γ·x̂ + β = 0·x̂ + 0 = 0 for every input. All activations collapse to zero, gradients vanish, and training stalls immediately. The standard initialization is γ = 1, β = 0 — this makes BN an identity transform initially (output = normalized input), allowing gradients to flow and training to begin normally. γ and β then drift from these starting points as needed.
08 A network uses BN throughout. You decide to also add dropout with p=0.5. Is this a good idea? Why or why not? ▼
Usually not — BN's regularization effect already provides implicit dropout-like noise, and combining them often hurts performance. BN adds stochasticity through batch-dependent normalization statistics. Adding dropout on top adds another source of random noise that interacts poorly: when neurons are randomly zeroed by dropout, the remaining neurons' activations change scale, but BN re-normalizes them — effectively undoing part of what dropout is trying to do. Several empirical studies (including the original ResNet paper) found that BN + no dropout outperforms BN + dropout in image classification.
09 Why does BN allow using a significantly higher learning rate than training without it? ▼
BN keeps gradient magnitudes in a stable range, eliminating the catastrophic gradient explosion or vanishing that would occur with large learning rates in unnormalized networks. Without BN, a large learning rate in a deep network causes the loss landscape's curvature to vary wildly across layers — some layers need tiny steps, others need large ones, but one learning rate applies everywhere. BN normalizes each layer's input distribution, making the effective learning surface for each layer more uniform. This means a larger global learning rate can be safely applied everywhere simultaneously. Ioffe and Szegedy report that BN allows learning rates 5–10× higher than they otherwise could have used.
