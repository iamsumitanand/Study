Backpropagation

The algorithm that teaches every neural network — by running the chain rule in reverse.

Every weight in GPT-4 was set by running this algorithm trillions of times. Not a variant of it. Not something inspired by it. This exact algorithm — backpropagation — computed how each of the 1.7 trillion parameters contributed to the model's error, then handed those numbers to gradient descent to fix them. Once you understand backpropagation, you understand how all of modern deep learning actually learns.

The Intuition

One idea: the chain rule, applied backwards

The entire idea of backpropagation is the chain rule from calculus, applied backwards through a network. Nothing more. The chain rule says: if A affects B and B affects C, then you can compute how A affects C by multiplying the two local rates of change. Backprop does this for every weight in the network — simultaneously, in a single backward pass.

Here is the mental model that makes it stick: imagine a network as a factory assembly line, where the final product is the loss (the error). Each worker (layer) blames the worker before them for how much they contributed to the defect. The blame flows backwards — from the loss, through the output layer, through the hidden layers, all the way to the first weights. At each step, each node multiplies the incoming blame by its own local rate of change, and passes the result upstream.

This "messenger carrying blame upstream" is backpropagation. The blame is the gradient of the loss with respect to each weight. The local rate of change is the derivative of each operation. The multiplication is the chain rule. That's the whole algorithm.

  • The forward pass computes outputs and stores intermediate values at every node.
  • The backward pass propagates gradients right-to-left, applying the chain rule at each node.
  • The local gradient at each node is just the derivative of that node's operation.
  • The upstream gradient arrives from the node ahead in the backward pass.
  • The output gradient is local × upstream — passed further backward.

Understanding It Deeply

Build it from first principles — don't just accept it

Backpropagation works on any computation that can be expressed as a directed acyclic graph (DAG) of differentiable operations. Each node computes a function of its inputs. Gradients flow backwards through the edges. The chain rule is the glue.

Think first A network has a node whose output is used by two different downstream nodes — a branching point. When gradients flow backwards and both downstream nodes send a gradient back to this node, what should happen to them? Think about what the chain rule says when a variable contributes to a loss through multiple paths.

The gradients from both paths are summed. This follows directly from the multivariate chain rule: if a variable x affects loss L through two paths, then ∂L/∂x = ∂L/∂a · ∂a/∂x + ∂L/∂b · ∂b/∂x. Backprop handles branching and merging naturally — branch points sum incoming gradients, merge points split them. This is why backprop works on arbitrarily complex graphs, not just linear chains.

Think first Early networks trained with sigmoid activations (σ(x) = 1/(1+e⁻ˣ)) famously failed to train deep architectures. The derivative of sigmoid is σ(x)·(1−σ(x)), which has a maximum value of 0.25. What happens when you multiply this repeatedly during backpropagation through 10 layers?

Each layer multiplies the gradient by at most 0.25. After 10 layers: 0.25¹⁰ ≈ 0.000001. The gradient shrinks exponentially as it travels backward — it effectively vanishes. Weights in early layers receive near-zero gradient signals and barely update. This is the vanishing gradient problem: deep networks trained with sigmoid activations couldn't learn because early layers saw no signal. It is the reason ReLU (whose gradient is exactly 1 for positive inputs) became universal in deep networks — it lets gradients pass through unchanged.

Think first To compute the gradient of a node during the backward pass, you need the derivative of that node's operation evaluated at the input it received. This means you need to know what the input actually was. Where does that information come from?

From the forward pass. Backprop cannot run without the forward pass having already run and stored its intermediate values. Consider a multiplication node z = x · y: its gradient with respect to x is y — you need the actual value of y that was computed during the forward pass. Modern autodiff systems like PyTorch store these "saved tensors" in the computation graph during the forward pass so the backward pass can retrieve them. This is why training a large network requires memory proportional to the depth — you must cache every intermediate activation.

The Math

Chain rule + a 3-node worked example

The chain rule is the only calculus you need. For a composition L = f(g(w)), the derivative of L with respect to w is:

Chain Rule (General Backprop Update)
∂L/∂w = ∂L/∂z · ∂z/∂w
∂L/∂w the gradient we want — how much loss changes when we nudge weight w ∂L/∂z upstream gradient — how much loss changes when z (the output of this node) changes; arrives from the layer ahead ∂z/∂w local gradient — derivative of this node's operation with respect to w; computed analytically for each op type

Full worked example: 2 inputs, 1 hidden unit, 1 output — with exact numbers.

Network: x₁=2, x₂=3, weights w₁=0.5, w₂=−1.0, w₃=0.8. Hidden unit uses sigmoid activation. Output is a linear node. Loss = squared error vs target t=1.0.

Forward Pass

1 Weighted sum into hidden unit: z₁ = w₁·x₁ + w₂·x₂ = 0.5·2 + (−1.0)·3 = 1 − 3 = −2.0
2 Sigmoid activation: a₁ = σ(z₁) = σ(−2.0) = 1/(1+e²) ≈ 0.1192
3 Output (linear): ŷ = w₃·a₁ = 0.8 × 0.1192 ≈ 0.0954
4 Loss (squared error): L = (ŷ − t)² = (0.0954 − 1.0)² ≈ 0.8194

Backward Pass

1 Gradient of loss w.r.t. output: ∂L/∂ŷ = 2(ŷ − t) = 2(0.0954 − 1.0) ≈ −1.809
2 Gradient w.r.t. w₃ (chain rule at output): ∂L/∂w₃ = ∂L/∂ŷ · ∂ŷ/∂w₃ = −1.809 × a₁ = −1.809 × 0.1192 ≈ −0.216
3 Gradient through to a₁ (chain rule at output): ∂L/∂a₁ = ∂L/∂ŷ · w₃ = −1.809 × 0.8 ≈ −1.447
4 Gradient through sigmoid (local grad = σ·(1−σ)): ∂L/∂z₁ = −1.447 × σ(z₁)(1−σ(z₁)) = −1.447 × 0.1192 × 0.8808 ≈ −0.1519
5 Gradient w.r.t. w₁: ∂L/∂w₁ = ∂L/∂z₁ · x₁ = −0.1519 × 2 ≈ −0.3038
6 Gradient w.r.t. w₂: ∂L/∂w₂ = ∂L/∂z₁ · x₂ = −0.1519 × 3 ≈ −0.4557
All three gradients computed in one backward pass. Gradient descent will now update: w₁ += −α·(−0.3038), w₂ += −α·(−0.4557), w₃ += −α·(−0.216). All negative gradients — meaning all weights should increase to push the prediction toward the target 1.0. The math agrees with intuition.

From Theory to Practice

How autodiff libraries implement this

No one implements backpropagation by hand for modern networks. Instead, frameworks like PyTorch and JAX use automatic differentiation (autodiff) — they build a computation graph dynamically and then differentiate it exactly, using the same chain rule rules, but applied automatically to any code you write.

In PyTorch, every tensor operation records itself in a computation graph. When you call loss.backward(), PyTorch traverses this graph in reverse topological order, applying the chain rule at each node. Each operation (add, multiply, sigmoid, ReLU, etc.) has a registered backward() function that computes its local gradient. The upstream gradient arrives as an argument; the function multiplies it by the local gradient and returns the result to be passed further back.

JAX takes a different approach: it transforms functions rather than recording graphs. jax.grad(f) returns a new function that, when called with an input, computes the gradient of f at that input. This is "source transformation" autodiff — JAX analyzes the Python function at trace time and generates gradient code. Both approaches compute identical results; they differ only in how they represent and traverse the computation graph.

The computational graph perspective: In both frameworks, "the network" is not a fixed architecture — it is the computation graph produced by one forward pass. This means graphs can change shape depending on the input (dynamic graphs), enabling architectures like variable-length RNNs and conditional computation. PyTorch's "define-by-run" model builds a new graph on every forward pass, which is why debugging deep learning code in PyTorch feels like debugging ordinary Python.

Common Misconceptions

Five causal misconceptions — and why they matter

These aren't isolated errors. Each one is a consequence of misunderstanding the previous. Trace the chain and all five become impossible to hold simultaneously.

  • ① Backpropagation trains the network.
    Backpropagation only computes gradients — it never changes a single weight. Gradient descent (or Adam, RMSProp, etc.) uses those gradients to update weights. Calling backprop "the training algorithm" is like calling a thermometer "the fever reducer." Both are required; they do fundamentally different things.
  • ② Backpropagation propagates errors backward.
    It propagates gradients — specifically, the gradient of the loss with respect to each weight. Gradients are local slope measurements, not error signals. The distinction matters: a large gradient doesn't mean a large error, and a small gradient doesn't mean a small error. You can have enormous errors with tiny gradients (the vanishing gradient problem).
  • ③ The forward pass and backward pass use different data.
    The backward pass is completely dependent on the forward pass. Every local gradient requires the intermediate activation values computed in the forward pass. This is why backpropagation must store all intermediate values in memory — the backward pass will need them. Training a 100-layer network requires holding 100 activation tensors in GPU memory simultaneously.
  • ④ Backprop is specific to neural networks.
    Backpropagation is just reverse-mode automatic differentiation applied to a computation graph. Any differentiable computation — physics simulators, rendering engines, differentiable programming in general — can use the same algorithm to compute gradients. Neural networks are the most common application, but the algorithm is general.
  • ⑤ Deeper networks always converge faster because backprop runs more efficiently with more layers.
    The opposite is often true. Deeper networks amplify both the vanishing gradient problem (gradients shrink through sigmoid layers) and the exploding gradient problem (gradients grow through layers with large weights). Each added layer multiplies the gradient by another number — if that number is consistently less than 1, gradients vanish; greater than 1, they explode. Techniques like batch normalization, residual connections, and careful weight initialization exist specifically to counteract this.

Summary

Five things to lock in

  • Backpropagation computes gradients using the chain rule in reverse through the computation graph. It changes nothing — it only measures.
  • The forward pass must run first and store its intermediate values. The backward pass retrieves those values to compute local gradients.
  • At each node: output gradient = local gradient × upstream gradient. At branching nodes, incoming gradients are summed (multivariate chain rule).
  • Vanishing gradients occur when local gradients are consistently less than 1 (e.g., sigmoid). Each layer multiplies the signal — 10 layers of 0.25 = signal × 10⁻⁶. ReLU solves this with a local gradient of exactly 1 for positive inputs.
  • Modern frameworks (PyTorch autograd, JAX) implement this automatically. You never write backprop by hand — you write the forward pass, and the framework differentiates it.
One question this chapter leaves unanswered: Backpropagation gives us the gradient of the loss with respect to every weight. Gradient descent says: update each weight by subtracting the gradient scaled by α. But what if the same learning rate α is too large for some weights and too small for others? What if the gradient is noisy and sends the optimizer in the wrong direction every other step?

That's why SGD is rarely used alone in modern training. The next chapter covers Optimizers — SGD with momentum, RMSProp, and Adam — each one a refinement of how gradients from backprop are used to update weights.

10 Questions

0 / 10 revealed

01 A node in a network receives two upstream gradients from two downstream nodes. How should they be combined before being passed further backward? ▼
They are summed. This is the multivariate chain rule: if a variable contributes to the loss through multiple paths, its total gradient is the sum of contributions from each path. ∂L/∂x = ∂L/∂a · ∂a/∂x + ∂L/∂b · ∂b/∂x. Backprop handles branching nodes naturally by accumulating gradients from all downstream paths before passing the result further backward.
02 You replace all sigmoid activations in a deep network with ReLU. Training immediately becomes more stable and gradients reach early layers. Explain why using the local gradient of each activation. ▼
ReLU's local gradient is 1 for positive inputs, which prevents gradient shrinkage. Sigmoid's local gradient is σ(1−σ) ≤ 0.25. Multiplying by 0.25 at every layer causes exponential decay (vanishing gradients). ReLU outputs max(0, x), so its derivative is 1 when x > 0 — the upstream gradient passes through unchanged. Multiplying by 1 repeatedly does not shrink the signal. Early layers receive gradients of similar magnitude to the output layer.
03 You call loss.backward() in PyTorch but forgot to call the forward pass first. What happens and why? ▼
The backward pass requires values from the forward pass to compute local gradients. Each backward function for an operation (e.g., multiplication) needs the values of its inputs from the forward pass — ∂(x·y)/∂x = y requires y. PyTorch stores these as "saved tensors" during the forward pass. Without running the forward pass, there is no computation graph to traverse and no saved activations to retrieve. In practice, PyTorch will raise an error.
04 A multiply gate has inputs x = 3 and y = −2. The upstream gradient arriving from the next layer is ∂L/∂z = 4. What is ∂L/∂x and ∂L/∂y? ▼
∂L/∂x = −8, ∂L/∂y = 12. For z = x·y: local gradient of z with respect to x is y, and with respect to y is x. Applying the chain rule: ∂L/∂x = ∂L/∂z · ∂z/∂x = 4 · y = 4 · (−2) = −8. ∂L/∂y = ∂L/∂z · ∂z/∂y = 4 · x = 4 · 3 = 12. The multiply gate acts as a "gradient switcher" — each input's gradient is the other input scaled by the upstream gradient.
05 You're training a 20-layer network and notice the loss stops decreasing after epoch 2, even though it's not overfit. The gradients for layers 1–5 are all extremely close to zero. What is happening and what are two ways to fix it? ▼
Vanishing gradients. The gradient signal shrinks as it propagates back through many layers. Two fixes: 1. Replace sigmoid/tanh with ReLU — ReLU's local gradient is 1 for positive inputs, so it doesn't shrink the signal. 2. Add residual connections (skip connections) — as in ResNets, they create shortcut paths that allow gradients to flow directly from the loss to early layers, bypassing the multiplicative attenuation of intermediate layers.
06 Why does training a deep network require GPU memory proportional to its depth, even if you only care about the output layer? ▼
The backward pass needs all intermediate activations from the forward pass. To compute the local gradient at layer k, you need the activation values that were input to layer k during the forward pass. With 100 layers, you must store 100 activation tensors simultaneously — one per layer — so the backward pass can retrieve them. This is called "activation memory." Gradient checkpointing is a technique that recomputes activations during the backward pass (at extra compute cost) to reduce peak memory.
07 An add gate has inputs a = 5 and b = −3, and upstream gradient ∂L/∂z = 2. What are ∂L/∂a and ∂L/∂b? ▼
Both equal 2. For z = a + b: ∂z/∂a = 1 and ∂z/∂b = 1. So ∂L/∂a = ∂L/∂z · 1 = 2 and ∂L/∂b = ∂L/∂z · 1 = 2. An add gate distributes the upstream gradient equally to both inputs — it acts as a "gradient distributor." This is distinct from a multiply gate, which routes each input's gradient through the other input's value.
08 Is backpropagation specific to neural networks? Name one other domain where it is routinely used. ▼
No — backpropagation is general reverse-mode autodiff applied to any differentiable computation graph. One example: differentiable rendering in computer graphics. Given a rendered image and a target image, you can backpropagate through the rendering equations to compute how to change 3D geometry, lighting, or material properties to match the target. Physics simulation is another — researchers backpropagate through physics simulators to optimize control policies or material parameters.
09 You double the weights in a network. Everything else is identical. What effect does this have on the gradients computed by backpropagation, and why? ▼
Gradients can grow — potentially causing exploding gradients. Consider a linear layer: its local gradient with respect to inputs involves the weight matrix. If weights are doubled, local gradients at each layer are roughly doubled. Through 10 layers, gradients scale by 2¹⁰ = 1024. This is the exploding gradient problem, the mirror image of vanishing gradients. Weight initialization strategies (Xavier/Glorot, He) are designed to keep gradients at roughly unit scale across layers during the backward pass.
10 PyTorch builds a new computation graph on every forward pass ("define-by-run"). JAX traces functions to generate gradient code. Both claim to implement backpropagation. Are they running the same algorithm? ▼
Yes — both implement reverse-mode automatic differentiation, which is exactly backpropagation. They differ in implementation, not in the mathematical algorithm. PyTorch records operations dynamically in a tape (eager mode), then traverses it in reverse. JAX traces the function symbolically and generates differentiated code via source transformation. Both compute identical gradients for identical inputs — the chain rule applied in reverse through the computation graph. The implementation differences affect debugging experience, performance characteristics, and support for dynamic control flow, not the algorithm itself.
