Calculus & Derivatives

The microscope that lets AI see which way to improve.

← Vectors & Linear Algebra Probability & Statistics →
A derivative answers one question: if I change this input by a tiny amount, how much does the output change? Every AI training loop asks this question billions of times. The answer is what tells the model which direction to improve — and by exactly how much.

The Intuition

Zoom in until it's a straight line

A derivative is a slope — the slope of a function at a single point. But what does "slope at a point" mean for a curve? The key insight is this: zoom in far enough on any smooth curve, and it looks like a straight line. The slope of that line is the derivative.

Formally, the derivative captures how much the output f(x) changes when you nudge the input by a small amount h: take the ratio of output-change to input-change, then shrink h to zero. The function's rate of change at that point is what you're left with.

This is the slope-as-rate-of-change interpretation: the derivative at x tells you approximately how much f changes per unit change in x — locally, right at that point.

  • If f'(x) > 0, the function is increasing — output goes up when input goes up.
  • If f'(x) < 0, the function is decreasing — output goes down when input goes up.
  • If f'(x) = 0, the function is momentarily flat — a critical point.

Partial derivatives extend this idea to functions of many variables. If your function is f(x, y, z), the partial derivative with respect to x asks: how does f change if I nudge x, while holding y and z completely frozen? You're differentiating in one direction at a time — treating all other variables as constants. This is the exact operation that backpropagation uses on neural network weights: one weight at a time, all others frozen.

Understanding It Deeply

Three questions worth sitting with

Think first What does it mean to take the derivative with respect to a matrix? A neural network layer maps an input vector to an output vector via a weight matrix W. When you want to know how the loss changes as you vary W, you're taking a derivative with respect to a matrix. What shape should that derivative be?

It should be the same shape as W itself. The derivative of a scalar loss L with respect to a matrix W is a matrix of the same dimensions — every entry (∂L/∂W)ij tells you how L changes if you nudge the single weight at row i, column j. This matrix of partial derivatives is the gradient of L with respect to W, and it has the same shape as W precisely so that the gradient descent update rule W := W − α · ∂L/∂W is dimensionally consistent.

Think first Why does the chain rule work — what is it actually doing? If y = g(f(x)), the chain rule says dy/dx = (dy/du)(du/dx) where u = f(x). This looks like fractions canceling. Is that a coincidence or the real explanation?

It isn't a coincidence, but fraction-cancellation isn't the real mechanism either — derivatives aren't actually fractions. What the chain rule is really saying is this: the composite function g(f(x)) amplifies changes in two steps. A nudge to x produces a change in u = f(x) by the factor du/dx. That change in u then propagates through g, amplified again by dg/du. The total amplification is the product. This is why backpropagation in a deep network multiplies gradients layer by layer — each multiplication is one application of the chain rule, propagating the error signal backward through a composed function.

Think first Why is the gradient a vector and not a scalar? If a function f(x, y) has a single scalar output, you might expect a single scalar to describe its rate of change. Why do we instead collect all the partial derivatives into a vector?

Because rate of change depends on direction. In a multi-variable function, the function can change at different rates along different directions. The gradient vector ∇f collects all the directional information: its components are the partial derivatives in each coordinate direction, and together they encode the direction and magnitude of steepest ascent in the input space. The gradient isn't one rate of change — it's the complete description of how the function changes in any direction. Moving in the direction of −∇f is what descends the steepest slope, which is exactly the update gradient descent makes.

The Math

From definition to training update

Six building blocks. Once you have them, the gradient descent update rule and backpropagation both fall out as direct consequences.

Derivative Definition (Limit)
f'(x) = limh→0 [f(x + h) − f(x)] / h
h infinitesimally small nudge to the input f(x+h)−f(x) resulting change in the output lim h→0 the ratio as the nudge shrinks to zero — the instantaneous rate of change
Power Rule
d/dx [xn] = n · xn−1
n any real exponent — the rule applies even to fractional and negative powers
Chain Rule
d/dx [g(f(x))] = g'(f(x)) · f'(x)
f(x) inner function — applied first g(u) outer function — applied to the result of f g'(f(x)) derivative of g, evaluated at the output of f — "how sensitive is g to its input, at the current input value?" f'(x) derivative of f — "how sensitive is f to x?"
Partial Derivatives
∂f/∂x = derivative of f treating all other variables as constants
∂ Leibniz notation for partial differentiation — distinguishes from total derivative d f(x,y) function of multiple variables; ∂f/∂x freezes y, ∂f/∂y freezes x
Gradient ∇f
∇f = [∂f/∂x₁, ∂f/∂x₂, …, ∂f/∂xₙ]
∇f vector of all partial derivatives — points in the direction of steepest ascent −∇f negative gradient — the direction gradient descent moves
Jacobian
Jij = ∂fi/∂xj
J matrix of all first-order partial derivatives of a vector-valued function f: ℝⁿ → ℝᵐ shape m × n — one row per output, one column per input; generalises gradient to vector outputs

Worked example: gradient of the loss for linear regression — and the training update

1 Define the loss. For a single training example with prediction wx + b and true label y, the squared loss is: L = (y − wx − b)²
2 Compute ∂L/∂w. Let the error e = y − wx − b. By the chain rule: ∂L/∂w = 2e · ∂e/∂w = 2e · (−x) = −2(y − wx − b)x
3 Compute ∂L/∂b. By the same chain rule, ∂e/∂b = −1: ∂L/∂b = 2e · (−1) = −2(y − wx − b)
4 Write the gradient descent update. Subtract the scaled gradient from each parameter. This is the exact step every training loop executes: w := w − α · (−2(y − wx − b) · x)
b := b − α · (−2(y − wx − b)) Calculus produces the gradient; gradient descent uses it to update weights. These two steps — differentiate, then step — are the complete training loop.

From Toy to Real

Automatic differentiation: calculus at scale

Manual differentiation works for two parameters. A large neural network has hundreds of millions. Nobody writes out partial derivatives by hand — PyTorch and JAX do it automatically using a technique called automatic differentiation (autograd).

The key idea is the computation graph. Every forward pass builds a directed acyclic graph of operations — multiply, add, activate, sum, etc. — where each node stores the operation performed and the local partial derivative of its output with respect to its inputs. This is just the chain rule, pre-assembled.

When you call loss.backward(), PyTorch traverses this graph in reverse, multiplying partial derivatives along each path from the loss back to the parameters:

Backprop as Chain Rule on the Computation Graph
∂L/∂w = (∂L/∂out) · (∂out/∂h) · (∂h/∂w)
∂L/∂out gradient of loss with respect to final output — seed value (= 1 for scalar loss) ∂out/∂h local Jacobian at the output layer — stored during forward pass ∂h/∂w local Jacobian at the hidden layer — stored during forward pass

This mechanical application of the chain rule across a stored graph is what makes deep learning computationally feasible. Each local partial derivative is cheap to compute (often just a multiplication by a constant or the input value), and the chain of multiplications gives the full gradient of the loss with respect to every parameter in one backward pass.

PyTorch's requires_grad=True flag tells the graph to track operations on a tensor. loss.backward() triggers the reverse traversal. .grad on each parameter contains ∂L/∂w — which is exactly the partial derivative the chain rule computed. The whole training loop is calculus made mechanical.

Common Misconceptions

Five wrong beliefs — and the truth behind each

These aren't independent mistakes. Each one builds on the previous. Trace the chain and they all dissolve together.

  • ① The derivative is the slope of the tangent line.
    Partially correct — but the derivative defines the slope of the tangent, it isn't defined by it. The derivative is a limit: the ratio of output change to input change as the input change shrinks to zero. The tangent line is just the geometric interpretation. What matters in ML is the value: how much does the loss change per unit change in this weight?
  • ② The chain rule is just multiplying derivatives together.
    Multiplying is the mechanism, but the meaning matters. The chain rule says: when functions compose, their local sensitivities multiply. The outer function's sensitivity (at the current inner output) times the inner function's sensitivity to its input. Forgetting the evaluation point — plugging in g'(x) instead of g'(f(x)) — is a common error that breaks backpropagation silently.
  • ③ The gradient and the derivative are the same thing.
    For a scalar function of one variable, yes — they coincide. But the gradient is a vector: it collects all partial derivatives of a scalar function of many variables. A derivative tells you the rate of change along one axis. The gradient tells you the rate of change in every direction simultaneously, and points in the direction of steepest ascent. In ML, every weight update uses the gradient (a vector or matrix), not a scalar derivative.
  • ④ Second derivatives don't matter for ML — gradient descent only uses the first.
    Gradient descent uses only the first derivative. But second derivatives (the Hessian) determine the curvature of the loss surface, which governs how well a given learning rate works. High curvature → the gradient-only linear approximation becomes inaccurate quickly → learning rate must be small. Low curvature → you can take larger steps. Second-order optimizers (Newton's method, L-BFGS) exploit the Hessian to take better steps. Adam implicitly adapts to curvature using gradient history.
  • ⑤ Differentiation requires a closed-form mathematical expression.
    Symbolic differentiation requires one — you need to manipulate the formula. But automatic differentiation (what PyTorch uses) operates on the execution trace of the computation, not the formula. It applies the chain rule to every elementary operation in the computation graph, regardless of whether the overall function has a clean closed form. This is why you can differentiate through control flow, loops, and arbitrary code — autograd doesn't need to know the formula, only what operations were executed.

Summary

Six things to lock in

  • The derivative is the limit of output-change/input-change as the nudge shrinks to zero — the instantaneous rate of change, not just a slope.
  • Partial derivatives differentiate in one direction while freezing all others — exactly what backprop does for each weight.
  • The gradient ∇f is a vector of all partial derivatives — same shape as the input, pointing in the direction of steepest ascent.
  • The chain rule multiplies local sensitivities across composed functions — this is what backpropagation applies mechanically across the computation graph.
  • The Jacobian generalises the gradient to vector-valued functions — it's an m×n matrix of partial derivatives, the full linear map between input and output changes.
  • Autograd builds a computation graph during the forward pass and applies the chain rule in reverse — you never write partial derivatives by hand for large models.
One question this chapter leaves unanswered: Calculus tells you how a model's output changes with its inputs — but which outputs should we care about? Training a model requires defining what "correct" means, and that requires probability. How do you turn "the model assigned probability 0.3 to the right answer" into a loss function? And why should you maximize log-probability rather than raw probability? That's where calculus hands off to probability theory.

9 Questions

01 f(x) = 3x⁴ − 2x² + 5. What is f'(x)? Then evaluate f'(1). ▼
f'(x) = 12x³ − 4x; f'(1) = 8. By the power rule: d/dx[3x⁴] = 12x³ and d/dx[−2x²] = −4x; the constant 5 vanishes. At x = 1: f'(1) = 12(1) − 4(1) = 8. This means the function is increasing at x = 1 at a rate of 8 units of output per unit of input.
02 h(x) = sin(x²). Use the chain rule to find h'(x). ▼
h'(x) = 2x · cos(x²). Let f(x) = x² (inner) and g(u) = sin(u) (outer). Then g'(u) = cos(u) and f'(x) = 2x. Chain rule: h'(x) = g'(f(x)) · f'(x) = cos(x²) · 2x. Note: the outer derivative is evaluated at the inner function's output (x²), not at x.
03 f(x, y) = x²y + 3y². Compute ∂f/∂x and ∂f/∂y. ▼
∂f/∂x = 2xy; ∂f/∂y = x² + 6y. For ∂f/∂x: treat y as a constant — d/dx[x²y] = 2xy, d/dx[3y²] = 0. For ∂f/∂y: treat x as a constant — d/dy[x²y] = x², d/dy[3y²] = 6y. The partial derivatives are the components of the gradient ∇f = [2xy, x² + 6y].
04 The gradient of a loss L at the current weights is ∇L = [3, −1, 2]. Learning rate α = 0.1. Write the gradient descent update for the three weights [w₁, w₂, w₃] = [0.5, 1.2, −0.3]. ▼
[0.2, 1.3, −0.5]. Each weight: wᵢ := wᵢ − α · (∂L/∂wᵢ). So: w₁ = 0.5 − 0.1·3 = 0.2; w₂ = 1.2 − 0.1·(−1) = 1.3; w₃ = −0.3 − 0.1·2 = −0.5. Notice w₂ increases (negative gradient means decreasing L means move in positive direction), while w₁ and w₃ decrease.
05 A neural network layer computes z = Wx + b, then a = σ(z). You want ∂L/∂W for gradient descent. Walk through the chain rule steps to get there from ∂L/∂a. ▼
∂L/∂W = (∂L/∂a) · (∂a/∂z) · (∂z/∂W). Step by step: ∂a/∂z = σ'(z) (derivative of activation function evaluated at z). ∂z/∂W = xᵀ (input transposed — standard linear layer Jacobian). So ∂L/∂W = (∂L/∂a) · σ'(z) · xᵀ. In code: this is exactly what PyTorch computes when you call loss.backward() — each factor is a local Jacobian stored during the forward pass.
06 f(x) = x³ − 2x² + 1. At x = 0, the derivative is 0. Is this a minimum, maximum, or neither? How can you tell? ▼
It is a local maximum. f'(x) = 3x² − 4x = x(3x − 4). At x = 0: f'(0) = 0 (critical point confirmed). Second derivative: f''(x) = 6x − 4; f''(0) = −4 < 0. Negative second derivative means concave down at this point → local maximum. The other critical point is at x = 4/3, where f''(4/3) = 6(4/3) − 4 = 4 > 0 → local minimum.
07 Why does PyTorch's autograd need to store intermediate values during the forward pass? What would go wrong if it didn't? ▼
Because chain rule requires local Jacobians evaluated at the forward-pass activations. The chain rule ∂L/∂w = (∂L/∂out) · (∂out/∂h) · (∂h/∂w) requires each local factor to be evaluated at the specific inputs that were computed during the forward pass. For example, ∂out/∂h at a sigmoid layer is σ(h)·(1−σ(h)) — this requires the value of h that was actually computed. Without storing it, the backward pass would need to recompute the entire forward pass, doubling memory or compute. Storing activations is the standard trade-off (gradient checkpointing trades compute for less memory).
08 A function maps ℝ³ → ℝ². What is the shape of its Jacobian? How many partial derivatives does it contain? ▼
Shape 2×3; contains 6 partial derivatives. The Jacobian Jᵢⱼ = ∂fᵢ/∂xⱼ has one row per output dimension (2) and one column per input dimension (3). Each entry is one partial derivative: how does the i-th output change when you nudge the j-th input? The 2×3 Jacobian is the linear map that best approximates the function locally — it generalises the gradient (which is a special case: 1 output → 1×n Jacobian = row vector = gradient transposed).
09 You compute the gradient of a loss and find that ∂L/∂w = 0 for all weights. Does this mean the model is fully trained? ▼
Not necessarily — a zero gradient means a critical point, not necessarily a good solution. The gradient is zero at local minima, local maxima, and saddle points. In high-dimensional spaces (many weights), saddle points are exponentially more common than local minima. The model could be stuck at a saddle where loss is still high. Additionally, "fully trained" requires good generalisation — even if you've found a true minimum of the training loss, the model may overfit. Training loss minimum ≠ good test performance.
