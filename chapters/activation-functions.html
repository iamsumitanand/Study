Part 2 · Neural Networks

Activation Functions

The single ingredient that turns a stack of matrix multiplications into a learning machine.

Remove the activation function and your 100-layer network is secretly just one linear layer. Every weight matrix collapses into a single equivalent matrix — no matter how deep you go. Non-linearity isn't a detail. It's the entire point.

The Intuition

Non-linearity is not a trick — it's what depth means

A linear transformation can only rotate, scale, and shear. No matter how many you stack, the composition is still just one linear transformation. Activation functions break this by introducing a bend — after each linear step, the space gets twisted in a new way. That twist is what makes depth meaningful.

Think of the layers as a transformation hierarchy. Early layers bend the input space so that simple patterns — edges, tone shifts, character strokes — become linearly separable at the next level. Middle layers combine those separable patterns into shapes, phonemes, or semantic concepts. Late layers assemble those into the final decision. Without activation functions, none of this hierarchy exists — you get one flat linear map from input to output.

This is why the Universal Approximation Theorem says a single hidden layer with enough neurons and a nonlinear activation can approximate any continuous function. The activation function is what buys that expressive power. Without it, you can only approximate linear functions — regardless of width or depth.

  • The pre-activation z = Wx + b is always linear — a weighted sum of inputs.
  • The activation function σ(z) applies a nonlinear squeeze, gate, or threshold.
  • The output a = σ(z) becomes the input to the next layer — now living in a twisted space.
  • The gradient of the activation at each neuron controls how much signal backpropagates through it.

Understanding It Deeply

Three questions that reveal the whole story

Think first Sigmoid squashes any input into (0, 1). That sounds useful. But when |z| is large — say z = 10 or z = −10 — the output is nearly 0 or nearly 1. What does that do to the gradient, and why does it cause problems in deep networks?

The sigmoid derivative is σ(z)(1 − σ(z)). At saturation — when z is far from zero — both σ(z) and (1 − σ(z)) are close to 0 or 1, so their product approaches zero. The gradient nearly vanishes. In backpropagation, gradients are multiplied through every layer in the chain rule. With even 5 layers, a gradient that starts at 0.01 per layer becomes 0.01⁵ = 10⁻¹⁰ — effectively zero. The earliest layers receive no training signal. This is the vanishing gradient problem, and sigmoid's saturation regions are its primary cause in early deep networks.

Think first ReLU is defined as max(0, x). Nothing clever about that — it just zeroes out negative inputs. Why does this absurdly simple function outperform sigmoid in deep networks, and what does the gradient look like everywhere?

ReLU's gradient is exactly 1 for any positive input and 0 for any negative input. There is no saturation region on the positive side — no matter how large z gets, the gradient never shrinks. This means the chain rule product through many layers stays at 1 (for positive activations), and early layers receive full gradients. ReLU is also cheap to compute — one comparison, no exponentials — and sparse by nature: roughly half the neurons in a typical network output zero at any given time, which acts as implicit regularization. The simplicity is the point. It solves the vanishing gradient problem where sigmoid fails.

The cost is the dying ReLU problem: if a neuron's pre-activation is consistently negative (e.g., from a large negative bias), its gradient is permanently zero. The neuron stops learning entirely and can never recover. Leaky ReLU, ELU, and related variants address this by allowing a small negative slope (e.g., 0.01x) rather than a hard zero.

Think first Researchers used sigmoid for decades. Then ReLU became standard. Now transformers use GELU. Each replacement came with empirical evidence that the new function trained better. What structural property could make GELU better than plain ReLU in attention-based architectures?

ReLU is a hard gate: every negative input is exactly zeroed out. GELU (Gaussian Error Linear Unit) softens this — instead of a binary zero/one gate, it weights each input by the probability that a Gaussian random variable would exceed it. Near zero, GELU has a smooth stochastic behavior: inputs slightly below zero are still partially passed through. This smoothness is gradient-friendly, especially for the stochastic attention patterns in transformers, where many pre-activations cluster near zero. GELU's smooth gradient also reduces the risk of the dead neuron problem while keeping ReLU's non-saturating behavior for large positive inputs. Empirically, GELU consistently outperforms ReLU in transformer architectures (BERT, GPT, and successors all use it), though the advantage shrinks in simpler CNNs where ReLU remains competitive.

The Historical Arc

Sigmoid → Tanh → ReLU → GELU: why each replaced the previous

The history of activation functions is a sequence of practitioners hitting a wall, diagnosing the cause, and finding a function that removed it. Each transition is a case study in what "better" means.

Sigmoid (1980s–2000s) was the natural first choice. Its output maps to (0, 1), interpretable as a probability. Its smooth derivative seemed ideal for gradient descent. It worked well for shallow networks and single-layer perceptrons. But when researchers tried adding layers — hoping for more representational power — training became unreliable. Gradients shrank toward zero as they propagated backward through each sigmoid layer. Training the first layer was nearly impossible with five or more layers. The vanishing gradient problem was identified and named, but the solution wasn't obvious.

Tanh (RNNs, late 1990s–2010s) was an improvement for a specific reason: zero-centring. Sigmoid outputs are always positive, which means the gradients of the weights below a sigmoid layer are always the same sign. This causes a zig-zag update pattern in gradient descent — each step can only move in a restricted direction, slowing convergence. Tanh centres outputs around zero, eliminating this. It became the default for recurrent networks (LSTMs use tanh for cell state outputs) but still saturates and still causes vanishing gradients in very deep feedforward nets.

ReLU (deep CNNs, 2010–2017) solved the vanishing gradient problem in one stroke. Krizhevsky et al.'s AlexNet (2012) demonstrated that deep convolutional networks trained with ReLU converged six times faster than with tanh. The gradient of 1 for any positive input meant that no matter how many layers the backward pass traversed, the signal did not decay (for active neurons). ReLU also made networks sparse — neurons either fire or don't — which acted as implicit regularization and reduced overfitting. It became the universal default for CNNs and most feedforward architectures and remains the first thing to try today.

GELU (transformers, 2018–present) emerged from BERT and GPT. In these architectures, many pre-activations cluster close to zero — attention weights and residual stream values are often small. ReLU's hard zero-floor is problematic here: a large fraction of neurons die or sit exactly on the discontinuity. GELU's smooth, stochastic gate handles this more gracefully. The Gaussian weighting naturally down-weights small negative inputs without fully killing them, and it's differentiable everywhere. BERT, GPT-2, GPT-3, PaLM, LLaMA, and virtually every transformer architecture published after 2019 use GELU (or its close cousin SwiGLU).

The pattern is consistent: each new function was adopted not because it was theoretically superior in isolation, but because it removed the specific bottleneck that practitioners were hitting at the depth and architecture of the moment. Activation function choice is inseparable from network architecture. The same function that works poorly in one context can be essential in another.

Common Misconceptions

Five causal mistakes — and how to unlearn them

Each misconception listed here leads naturally to the next. Tracing the chain makes all five impossible to forget.

  • ① More layers automatically means more expressive power.
    Not without activation functions. Stacking linear layers without non-linearities between them is mathematically identical to a single linear layer — the composition W₂(W₁x) = (W₂W₁)x. Depth only buys representational power when each layer introduces non-linearity.
  • ② Sigmoid is the natural default because its output looks like a probability.
    Sigmoid is appropriate as the final output layer for binary classification, where probability interpretation matters. As a hidden layer activation in deep networks, its saturation causes vanishing gradients that cripple training. The probability interpretation is irrelevant in hidden layers — neurons don't need to output probabilities, they need to pass gradients.
  • ③ So use a non-saturating function like ReLU everywhere — it solves all gradient problems.
    ReLU solves vanishing gradients for positive inputs but introduces the dying ReLU problem for negative inputs. A neuron whose pre-activation is always negative has a gradient of exactly zero — it cannot update its weights and will never recover, no matter how long training runs. Large learning rates and poor weight initialisation can push many neurons into this state simultaneously, effectively reducing network capacity.
  • ④ Dead neurons are rare and a minor concern in practice.
    In poorly initialised networks or with aggressive learning rates, dead neuron fractions of 10–40% are reported. These neurons contribute nothing to forward pass outputs and nothing to backward pass gradients. The network trains as if it had that many fewer neurons — but you're still paying the compute cost. Batch normalisation, careful weight initialisation (He initialisation for ReLU), and Leaky ReLU are all practical mitigations.
  • ⑤ GELU is just a smoother ReLU — the choice between them is cosmetic tuning.
    The smooth gradient near zero is architecturally significant in transformers, where pre-activations frequently cluster near zero due to residual connections and attention scaling. GELU's stochastic gating behaviour was empirically shown to improve BERT's accuracy by ~1% on GLUE benchmarks vs ReLU — a meaningful margin. The choice is not cosmetic; it's a design decision tied to the distribution of pre-activations in the specific architecture. For CNNs trained on ImageNet, the gap is much smaller.

Summary

Five things to lock in

  • Without activation functions, any depth of network collapses to a single linear transformation — depth is meaningless without non-linearity.
  • Sigmoid and tanh saturate: for large |x|, their derivatives approach zero. In deep networks, this zeroes out gradients in early layers — the vanishing gradient problem.
  • ReLU solves vanishing gradients for positive inputs (gradient = 1) but creates the dead neuron problem for permanently negative inputs (gradient = 0, unrecoverable).
  • GELU's smooth stochastic gate avoids both hard saturation and hard zeroing, making it the default in transformer architectures where pre-activations cluster near zero.
  • Activation function choice is architecture-specific, not universal: sigmoid is correct for binary output layers; tanh remains in LSTMs; ReLU is the right default for CNNs; GELU for transformers.
One question this chapter leaves unanswered: We now know that the gradient of the activation function at each neuron controls how much signal propagates backward. But exactly how does the network compute those gradients for every weight across every layer in a single pass? Doing it naively — recomputing from scratch for each weight — would cost O(n²) operations per step in a large network. Modern networks train in O(n). The algorithm that achieves this is backpropagation. And it's the next chapter.

Test Your Understanding

9 Questions

0 / 9 revealed

01 A 5-layer network uses no activation functions — just matrix multiplications. You double the number of layers to 10. Does the network's representational power increase? ▼
No. Without activation functions, any number of stacked linear layers is mathematically equivalent to a single linear layer: W₅ · W₄ · W₃ · W₂ · W₁ = W_combined. Adding more layers only changes the combined matrix — not what the network can represent. Representational power in neural networks comes from the non-linearities, not the depth itself.
02 Sigmoid outputs are always positive (range: 0 to 1). Why does this specifically slow down gradient descent, independent of the saturation problem? ▼
Zig-zag gradient updates. When all activations feeding into a weight are positive, the gradient of that weight is always the same sign as the upstream error signal. This means weights can only move in one direction per step — either all increase or all decrease simultaneously. To reach a minimum that requires mixed updates, the optimizer must zig-zag across the loss surface. Tanh avoids this by producing zero-centred outputs (range: −1 to 1), allowing gradients of mixed signs in the same step.
03 A neuron using ReLU has weights [−2, 3] and bias −5. For any typical input, what will happen to this neuron during training, and why? ▼
It will become a dead neuron and stop learning. The large negative bias (−5) makes the pre-activation z = w·x + b very likely to be negative for typical inputs. ReLU maps any negative z to exactly 0, and more critically, ReLU′(z) = 0 for z ≤ 0. With zero gradient, no weight update occurs during backpropagation. The neuron is stuck: it cannot change its weights to escape this state. This is the dying ReLU problem — the neuron is permanently silenced.
04 You have a 20-layer network where every layer's sigmoid activation receives pre-activations of approximately z = 2.0. Estimate whether training the first layer is practically feasible. ▼
Not feasible. σ(2.0) ≈ 0.88, so σ′(2.0) = 0.88 × 0.12 ≈ 0.105. Through 20 layers, the chain rule multiplies this 20 times: 0.105²⁰ ≈ 10⁻²⁰. This is numerically indistinguishable from zero in floating-point arithmetic. The first layer receives no training signal at all. This is the vanishing gradient problem in practice — the reason training deep networks with sigmoid was described as "practically infeasible" before ReLU.
05 Why is sigmoid still used at all, given its vanishing gradient problem? Name one specific scenario where it is the correct choice. ▼
Binary classification output layer. Sigmoid is correct as the final activation when the task requires a probability output in (0, 1), such as binary classification. The vanishing gradient problem only matters in hidden layers — the output layer only needs to produce the right value, not propagate gradients further. Using sigmoid here is mathematically natural: it pairs directly with binary cross-entropy loss and produces a well-calibrated probability. The problem arises only when sigmoid is placed in hidden layers of deep networks.
06 ReLU and GELU both output approximately the same value for large positive x. For what range of x do they differ most, and why does this matter for transformers? ▼
Near x = 0 (approximately −1 to 1). ReLU is a hard gate — exactly 0 for x ≤ 0, exactly x for x > 0, with a discontinuous derivative at 0. GELU smoothly transitions around 0, outputting a slightly negative value for small negative x (dipping to about −0.17 at x ≈ −0.5) and growing smoothly for positive x. In transformers, attention weights and residual stream values frequently cluster near zero. A hard gate at zero creates a large fraction of neurons that are either fully active or fully dead — GELU's smooth transition handles this distribution more gracefully, providing better gradient flow near the decision boundary.
07 Your CNN is training with ReLU and you notice that 35% of your neurons output zero for every sample in the training set. Should you be concerned? What might be causing this? ▼
Yes — this indicates a significant dead neuron problem. 35% of capacity is wasted. Likely causes: (1) Learning rate too large — weight updates pushed neurons into a permanently negative pre-activation region. (2) Poor weight initialisation — weights initialised too large can cause large pre-activations that with the wrong sign produce zero gradients immediately. (3) Large negative biases — explicit or emergent. Mitigations: use He initialisation (designed specifically for ReLU), lower the learning rate, add batch normalisation to stabilise pre-activations, or switch to Leaky ReLU which allows a small gradient (0.01x) for negative inputs.
08 Tanh is said to be "better than sigmoid" for hidden layers. But LSTM cells use tanh for the cell state output. Isn't that contradictory — using a saturating activation in a deep network? ▼
No — LSTMs solve the vanishing gradient problem through gating, not the activation function. An LSTM's cell state is updated through an additive recurrence: c_t = f·c_{t-1} + i·g. The forget gate f and input gate i regulate this additively, so gradients can flow through many time steps without being multiplied by a small derivative repeatedly. The tanh applied to the cell output is just for bounding the output range (−1, 1) before it enters the hidden state. The gradient highway in an LSTM bypasses this tanh via the additive cell state path — this is the architectural innovation, not the choice of tanh.
09 You're designing a new architecture for a vision task. You have no particular constraints. Which activation function should you start with in the hidden layers, and what single property makes it the right default? ▼
ReLU — because its gradient is exactly 1 for any positive input. No saturation, no gradient decay for positive activations. It is cheap to compute, well understood, and has been validated across thousands of vision architectures. The dead neuron problem is manageable with proper He initialisation and reasonable learning rates. GELU is worth considering for transformer-style or attention-heavy architectures where pre-activations cluster near zero, but for a standard CNN on vision tasks, ReLU is the correct first choice. Always start simple.
