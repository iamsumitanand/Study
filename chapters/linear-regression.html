Part 1 · Basic Machine Learning

Linear Regression

The model that started modern machine learning — and still powers much of it.

You have 500 past house sales — size, location, price. A new house hits the market: 1,450 sq ft, suburban. What will it sell for?

This is the canonical prediction problem, and linear regression is the oldest rigorous answer to it. Before neural networks, before random forests, before support vector machines — there was a line through data points. And remarkably, that line is still inside most of the complex models you use today. Understanding it fully means understanding the foundation everything else is built on.

The Intuition

Fitting a line through data — and why that's not trivial

Given a set of points on a chart, your eye naturally draws a line that "captures the trend." Linear regression formalizes exactly that instinct: find the line where the vertical distances from each data point to the line are collectively as small as possible.

The vertical distance from a point to the line is the residual — the error your model makes on that training example. You want all residuals to be small simultaneously. But here's the problem: you can't just minimize the sum of raw residuals. Some are positive (prediction too low), some negative (prediction too high), and they cancel out. A line that's wildly wrong can still have residuals that sum to zero.

The fix: square every residual before summing. Squaring makes all errors positive, penalizes large errors disproportionately, and creates a smooth, differentiable surface that gradient descent can navigate. This is Mean Squared Error (MSE) — the standard loss for linear regression — and squaring is not arbitrary. It has a deep probabilistic justification: minimizing MSE is equivalent to maximum likelihood estimation under the assumption that prediction errors are Gaussian (d2l.ai §3.1.3).

  • The model is a parametric function: ŷ = w·x + b — a slope and an intercept.
  • The loss is MSE — average of squared residuals across all training points.
  • The goal is to find w and b that minimize MSE on the training data.
  • There are two routes to the minimum: solve it analytically (OLS normal equation), or descend iteratively (gradient descent).

Understanding It Deeply

Three questions that force real understanding

Linear regression looks simple. But simple models have sharp edges — they reward careful thinking. Each question below points at a conceptual trap that catches students and practitioners alike.

Think first Why do we use squared error rather than absolute error? Both penalize wrong predictions. Both give zero for perfect predictions. List at least two concrete reasons to prefer squared error, and one reason absolute error might actually be better.
Show answer
Reasons to prefer squared error: (1) It's differentiable everywhere. Absolute error has a kink at zero — its derivative is undefined at the point of perfect prediction, which breaks gradient descent. (2) Squaring produces a convex, bowl-shaped loss surface with a unique global minimum — the math is cleaner and optimization is guaranteed to converge. (3) It has the maximum likelihood interpretation under Gaussian noise: if you believe errors come from a normal distribution, minimizing MSE is the statistically correct thing to do (d2l.ai §3.1.3).

When absolute error (MAE) is better: When your dataset has outliers. Squaring amplifies outliers — a point that's 10 units off contributes 100 to the loss, while absolute error contributes only 10. If a few bad data points exist (mislabeled, corrupted), MSE will warp the entire line to accommodate them. MAE is more robust.
Think first The OLS normal equation gives the exact optimal weights in one matrix operation: w* = (XTX)−1XTy. If this is exact and immediate, why does anyone bother with gradient descent for linear regression? Think about what happens when you have 10 million training examples and 500 features.
Show answer
Cost of matrix inversion. Inverting an (n×n) matrix costs O(n³) operations. With 500 features, XTX is a 500×500 matrix — fast to invert. But if you have 10 million training examples and the matrix isn't full-rank (features are correlated), inversion becomes numerically unstable or impossible.

The DL Book (§2.3) identifies this precisely: when the condition number of XTX is large (columns nearly linearly dependent), small errors in X get amplified enormously in the inverse — the normal equation becomes numerically unreliable. Gradient descent avoids inversion entirely. It also scales to cases where the normal equation doesn't apply at all: deep networks, non-differentiable losses, models far beyond linear. For linear regression on small, well-conditioned problems, use the normal equation. For everything else, gradient descent.
Think first You train a linear regression model on house prices. Training MSE is low. You deploy it and predictions are terrible. Your colleague says: "just add more features — bedroom count, distance to school, age of house." Will that fix it? What two fundamentally different problems could explain the gap between training and deployment performance?
Show answer
Problem 1 — Overfitting (generalization gap): The model memorized training data instead of learning the true pattern. Adding more features makes this worse, not better, unless you also add regularization. d2l.ai §3.6 calls this the central tension: training loss and generalization are not the same objective.

Problem 2 — Distribution shift: Your deployment data is structurally different from training data — different city, different economic conditions, post-pandemic market. No amount of feature engineering fixes this; the model is being asked to extrapolate beyond its training distribution. Adding bedroom count doesn't help if the relationship between bedrooms and price differs in the deployment context.

The Math

OLS normal equation and gradient descent update rules

There are exactly two ways to find optimal weights for linear regression. One is a direct algebraic formula. The other is the iterative process from the previous chapter. Both solve the same problem; which you use depends on the size and structure of your data.

OLS Normal Equation (d2l.ai §3.1.2)
w* = (XTX)−1 XTy
X design matrix — n rows (examples) × d cols (features), shape [n, d] y target vector — n true values, shape [n, 1] XTX d×d covariance matrix — must be invertible (full-rank, no perfectly correlated features) w* optimal weight vector — exact closed-form solution when XTX is well-conditioned
Gradient Descent Update Rules (d2l.ai §3.1.1)
w := w − (α/n) · Σ (ŷ(i) − y(i)) · x(i)
b := b − (α/n) · Σ (ŷ(i) − y(i))
α learning rate — step size, controls how aggressively we adjust each iteration n number of training examples — we average the gradient over all examples ŷ(i) − y(i) residual on example i — how far the current prediction is from the truth x(i) feature value of example i — weights the gradient contribution by input magnitude

A 4-step worked example — watch w converge toward the truth:

Setup: 3 houses — sizes 1000, 1500, 2000 sq ft — prices $200k, $280k, $380k.
True slope ≈ 0.18 ($180/sq ft). Start: w = 0.05, b = 0. Learning rate α = 0.0000002.

The gradient is large early (big corrections) and shrinks automatically as w improves — the same self-regulating property from gradient descent. After ~hundreds of iterations, w converges to 0.18 and b converges to ~20 ($20k base).

From Toy to Real

House price prediction — from 1D to the real world

Everything so far used one feature: house size. Real estate agents know that size alone is a poor predictor. The same 1,500 sq ft house sells for $200k in one zip code and $900k in another. Location matters. Age matters. Number of bedrooms matters. Garage. School district. So how does the model grow from one dimension to many?

The answer is elegant: almost nothing changes. The 1D model ŷ = w·x + b becomes the multi-dimensional model ŷ = w₁x₁ + w₂x₂ + ... + wₐxₐ + b, or in vector notation: ŷ = wTx + b. Each feature gets its own weight. The loss function (MSE) is identical. The gradient descent update rule runs the same way — just applied to each weight simultaneously.

d2l.ai frames it directly: the design matrix X collects all n training examples into an n×d matrix where d is the number of features. The target vector y is n×1. The normal equation w* = (XTX)−1XTy still applies. The prediction for any new house is just a dot product of its feature vector with the learned weights.

What breaks at scale: The DL Book (§2.3) identifies the danger in the normal equation at scale: the condition number of XTX. When features are correlated (living area and total rooms both grow together), XTX becomes ill-conditioned — small noise in the data gets magnified enormously in the matrix inverse, producing wildly unstable weights. This is why real regression pipelines include feature normalization, regularization (Ridge/Lasso), and often prefer gradient descent over the closed-form solution when dealing with many correlated features.

The practical workflow for a real house price model:

  • Collect features: size, bedrooms, bathrooms, zip code (one-hot), age, garage (binary).
  • Normalize each feature to zero mean, unit variance — prevents ill-conditioning.
  • Split data: training set to fit weights, validation set to detect overfitting, test set for final evaluation.
  • Fit via gradient descent (or OLS if small enough and well-conditioned).
  • Evaluate on the test set — not training MSE, which will always look good.

Common Misconceptions

Five wrong beliefs — and why they're connected

These aren't five random mistakes. They form a chain — each one leads to the next. Trace the chain and they all become impossible to forget.

  • ① "Linear" means the model is simple and limited.
    Linear means the output is a linear function of the parameters, not necessarily of the raw inputs. You can feed x², log(x), or x₁×x₂ as features — the model stays linear in its weights while capturing highly nonlinear relationships. Linear regression with polynomial features can fit curves, not just lines. "Linear" describes the model class, not the complexity of relationships it can represent.
  • ② So I can just throw in any features and the model will figure it out.
    This leads directly to overfitting. With enough features, a linear model can perfectly memorize any training set — and completely fail on new data. d2l.ai §3.6 emphasizes this: low training error is not the goal. Generalization is the goal. More features increase model capacity, which requires more data and regularization to prevent memorization.
  • ③ Once I've minimized training MSE, the model is done.
    Training MSE measures performance on data the model has already seen — it's optimistically biased. The DL Book §5.2 is explicit: if you evaluate only on training data, you have no way of knowing whether your model learned the true pattern or memorized the noise. Always evaluate on held-out test data. Training MSE and test MSE can diverge dramatically — a fitted model with 0 training MSE can have enormous test MSE if it memorized outliers.
  • ④ The residuals don't matter as long as MSE is low.
    Residuals are a diagnostic tool that reveals whether your model's assumptions are satisfied. Residuals should be: (1) approximately normally distributed, (2) homoscedastic (same variance across all predictions), (3) independent of predicted values. Systematic patterns in residuals — like a curve, or widening spread — mean your model is misspecified. Low MSE doesn't fix this. Inspecting residual plots is a non-negotiable step in applied regression.
  • ⑤ Linear regression tells you which features cause the outcome.
    Regression reveals association, not causation. A model that assigns high weight to umbrella sales for predicting rain has learned a correlation — it will make accurate predictions, but umbrellas don't cause rain. Without controlled experimentation or causal assumptions, regression weights measure the statistical relationship between features and outcome in the training data, not causal mechanisms. Confounders, reverse causality, and selection bias can all produce large regression coefficients that mean nothing causally.

Summary

Six things to lock in

  • The model: ŷ = wTx + b — a linear function of features, parameterized by weights w and bias b. Scales from 1D to thousands of dimensions without structural change.
  • The loss: MSE = (1/n) Σ (ŷ − y)². Squaring makes it differentiable, convex, and equivalent to maximum likelihood under Gaussian noise — not an arbitrary choice.
  • Two solutions: OLS normal equation (exact, O(d³), requires invertible XTX) vs gradient descent (iterative, scalable, no matrix inversion required).
  • Ill-conditioning: when features are correlated, XTX has a large condition number — small data noise explodes in the inverse. Normalize features and use regularization.
  • Training vs. generalization: low training MSE is necessary but not sufficient. The model must also generalize. Evaluate on held-out test data. Always.
  • Association ≠ causation. Regression coefficients measure statistical relationships in the training distribution — not causal effects.
One question this chapter leaves open: Linear regression predicts a continuous number. But most real prediction problems are about categories — spam or not spam, will this customer churn, will this patient have a heart attack. These outputs are not continuous. They're probabilities bounded between 0 and 1.

You could try to use linear regression for this — predict a number and threshold it. But linear regression will happily output values below 0 and above 1, and its loss function penalizes confident correct predictions. There's a better answer that shares nearly all of linear regression's structure but adds exactly one transformation to keep outputs in [0,1].

That's logistic regression. And it's the next chapter.

Test Your Understanding

9 Questions

0 / 9 revealed

01 You fit a linear regression with 2 features and get training MSE = 0. Does this mean you have a perfect model? ▼
Almost certainly no. A training MSE of 0 means the model perfectly memorized the training data — including any noise, outliers, or measurement errors. This is a sign of overfitting, not a good model. The question is always what MSE looks like on held-out test data. A model that perfectly fits 10 training points may predict the 11th terribly. d2l.ai §3.6: minimizing training error is not the goal — minimizing the gap between training and test error is.
02 Given 3 data points: (1, 2), (2, 4), (3, 5). Without computing anything, should the optimal regression line pass through all three points? Why or why not? ▼
It should not — and will not. The three points are not perfectly collinear (if they were, y would be 6 at x=3, not 5). Linear regression minimizes the sum of squared residuals, not the sum of residuals. A line through all three would require zero residuals everywhere, which is impossible with non-collinear data. The OLS solution finds the line where the total squared vertical distance is minimized — it will pass near but not through all three points. This is intentional: the line represents the best linear approximation to the true underlying trend.
03 What is the gradient of the MSE loss with respect to the bias b? (Express as a formula, then interpret it in plain English.) ▼
∂L/∂b = (1/n) · Σ (ŷ(i) − y(i)) — the average residual across all training examples. In plain English: if your model's predictions are systematically too high, the average residual is positive, so the gradient is positive, so the update subtracts from b (pushes predictions down). If predictions are systematically too low, gradient is negative, b increases. The bias term corrects the mean offset of your predictions. When b's gradient hits zero, your model predicts the right average — all remaining error is encoded in the weights w.
04 You have 1 million training examples and 10 features. Should you use the OLS normal equation or gradient descent? What changes if you have 1 million features? ▼
1M examples, 10 features: either works, normal equation is fine. XTX is only 10×10 — trivial to invert. The matrix operations scale with d³ (feature dimension cubed), not n (example count). The expensive part of the normal equation is computing XTX and XTy, which is O(nd) — linear in n.

1M features: use gradient descent. XTX is 1M×1M — inverting it is O(10¹⁸) operations, completely infeasible. Additionally, with that many features you almost certainly have correlated columns, making XTX ill-conditioned (DL Book §2.3). Gradient descent with regularization scales to high-dimensional problems that the normal equation cannot touch.
05 Why does minimizing MSE have a probabilistic justification? Under what assumption about errors does MSE become the "statistically correct" loss function? ▼
Under the assumption that prediction errors are i.i.d. Gaussian (normal). d2l.ai §3.1.3: if you model y = wTx + b + ε where ε ~ N(0, σ²), the likelihood of observing y given x is a Gaussian centered at wTx + b. Maximizing this likelihood (maximum likelihood estimation) over all n training examples is algebraically equivalent to minimizing the sum of squared residuals. So MSE is not just a convenient differentiable loss — it's the theoretically correct loss when you believe your noise is Gaussian. If noise is Laplacian (heavy-tailed), the correct loss is mean absolute error.
06 A linear regression model predicts ice cream sales using daily temperature. It gets weight w = 12.5 (sales per degree). Your manager says: "This proves hot weather causes people to buy more ice cream." What's wrong with this statement? ▼
The model proves correlation, not causation. w = 12.5 says that days with higher temperature are associated with more ice cream sales in the training data. It does not say that temperature causes the sales increase. A confounder — like "people go outside more in summer" — could explain both variables simultaneously. To establish causation, you'd need a controlled experiment (randomly assign temperature to days, which is impossible) or a causal inference framework with strong assumptions. Regression is a statistical tool; its coefficients are correlational unless the study design explicitly supports causal claims.
07 You plot the residuals of your fitted model and see a clear U-shaped pattern (residuals are positive at low and high predictions, negative in the middle). What does this tell you, and what should you do? ▼
The relationship is nonlinear — your model is misspecified. Residuals should be randomly scattered around zero with no pattern. A U-shape means the linear model systematically underpredicts at the extremes and overpredicts in the middle — there's a quadratic (or other curve) component in the true relationship that the linear model can't capture.

Fix: add polynomial features (x² term) to the design matrix. The model stays "linear regression" (linear in parameters) but can now fit a curve. Alternatively, if the relationship is truly complex, consider a nonlinear model. Always inspect residual plots before concluding your regression is well-specified.
08 Starting from w = 0.1, b = 0, with one data point (x=2, y=10), α = 0.1: what are w and b after one gradient descent step? Use MSE loss: L = (wx + b − y)². ▼
w = 0.772, b = 0.436.
Current prediction: ŷ = 0.1×2 + 0 = 0.2. Residual: ŷ − y = 0.2 − 10 = −9.8.
Loss gradient w.r.t. w: 2 × (ŷ − y) × x = 2 × (−9.8) × 2 = −39.2 (for single point, no 1/n).
Loss gradient w.r.t. b: 2 × (ŷ − y) = 2 × (−9.8) = −19.6.
Update: w = 0.1 − 0.1 × (−39.2) = 0.1 + 3.92 → Wait — this uses full gradient (not 1/n scaled).
With the (1/n) MSE convention: ∂L/∂w = (ŷ−y)×x = −9.8×2 = −19.6; w = 0.1 − 0.1×(−19.6) = 2.06. Either convention is valid — the key insight is: residual is large (−9.8), so the gradient is large, so the update is large. w moves dramatically toward the truth in one step because the error is enormous.
09 A colleague adds 50 new features to a house price model. Training MSE drops from 800 to 50. Test MSE goes from 850 to 3,200. What happened, and how would you fix it? ▼
Severe overfitting. The model with 50 new features memorized the training set but learned nothing generalizable. The 63× increase in test MSE is the signature of a model that has too many parameters relative to the information in the training data.

Fix options: (1) L2 regularization (Ridge): add λ·||w||² to the loss. This shrinks weights toward zero, penalizing complexity. (2) Feature selection: remove features that carry no real signal — test each feature's marginal contribution on a validation set. (3) Get more data: more training examples give the model more signal to learn from, reducing the ratio of parameters to data. (4) Cross-validation: use k-fold CV to select the feature set that minimizes validation MSE, not training MSE.
