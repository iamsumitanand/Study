Part 1 · Basic Machine Learning

Gradient Descent

The algorithm that trains every AI model you've ever heard of.

GPT-4, AlphaGo, Stable Diffusion, your spam filter — trained by one algorithm. Not different approaches for each. The same one. It's called Gradient Descent, and once you see how it works, you see the engine inside all of modern AI.

The Intuition

A search algorithm that follows the slope

CS229 frames it plainly: start with some initial guess for your parameter θ, then repeatedly change θ to make the loss J(θ) smaller, until you converge. That's gradient descent — a local search guided purely by the slope at your current position.

CS229 highlights one elegant property: the step is automatically proportional to the error. When your model is nearly correct, the gradient is small — tiny update. When you're far off, the gradient is large — big update. This falls out of the math; it isn't designed in.

  • The landscape is the loss function — a single number measuring how wrong the model is.
  • Your position is the model's current parameter (e.g., a weight w).
  • The slope at your position is the gradient — the direction that increases loss most steeply.
  • The step size is the learning rate α — how far you move each time.

Understanding It Deeply

Build it from first principles — don't just accept it

The Deep Learning Book derives gradient descent from one observation. The derivative f'(x) approximates how f changes for a small step: f(x + ε) ≈ f(x) + ε · f'(x). To decrease f, choose ε with the opposite sign to f'(x). That's the complete idea. The update rule x' = x − ε · ∇f(x) is just this made precise — the gradient points directly uphill, so we move directly downhill.

Think first The gradient tells you the slope at your current position. What does it say absolutely nothing about? Think of one scenario where following the downhill slope step-by-step would lead you somewhere bad.

It says nothing about global structure. You could be on the rim of a shallow local dip while a much deeper valley exists elsewhere. The gradient is purely local information. This is gradient descent's fundamental limitation — and why there is no mathematical guarantee it finds the global minimum.

When the gradient reaches zero, gradient descent stops. The DL Book calls these critical points. But a zero gradient can describe three distinct situations: a local minimum, a local maximum, or a saddle point — a point that is a minimum in some directions but a maximum in others. The algorithm has no way to tell them apart. It stops at all three.

Think first DL Book §8.2.3 proves that in high-dimensional spaces, saddle points are exponentially more common than local minima. If you're training a neural network with a million parameters and gradient descent stalls — what is it almost certainly stuck at?

A saddle point. And here's the counterintuitive part: the DL Book also shows that most local minima in large neural networks have acceptably low cost — they're fine solutions. The problem isn't bad local minima; it's saddle points where gradient ≈ 0 but the model is nowhere near a good solution. This is precisely why momentum-based optimizers like Adam exist: they accumulate velocity across steps and can push through flat saddle regions that pure gradient descent cannot escape.

Think first Gradient descent minimizes training loss. The DL Book states that training typically halts while the gradient is still large — not because optimization converged, but because validation error started rising. What does this tell you about what GD is actually optimizing for?

It optimizes training loss — and nothing else. Generalization to unseen data is a completely separate concern. In practice, the real stopping criterion isn't "gradient = 0"; it's "validation performance stopped improving." Optimization and learning are different objectives that happen to be coupled, but they can — and do — diverge.

The Math

The update rule

Plain English first: take your current parameter, subtract the gradient scaled by the learning rate. The gradient points uphill — subtracting it moves you downhill.

The Update Rule
w := w − α · (dL/dw)
w the parameter — what we're adjusting (a weight in the model) := assignment — overwrite w with the new value (not a test of equality) α learning rate — step size, a small positive number like 0.01 or 0.3 dL/dw gradient — how much loss L increases when we nudge w upward by a tiny amount

Where the gradient comes from — the geometry first. Our loss is L(w) = (w − 2)². Think of (w − 2) as the error — how far w sits from the minimum at 2. When error = 1, loss = 1. When error = 2, loss = 4. When error = 3, loss = 9. Loss grows as the square of the distance. So when you're already 2 units away and move 1 more, loss jumps by much more than when you're only 0.1 units away. The gradient dL/dw = 2(w − 2) captures exactly this: it's twice the current error. The factor of 2 is the signature of squaring — large error → large gradient → large corrective step. It's proportional punishment.

Four steps — watch the gradient shrink each time:

StepwGradientUpdateLoss
0 4.000 4.000 4 − 0.3×4 4.0000
1 2.800 1.600 2.8 − 0.3×1.6 0.6400
2 2.320 0.640 2.32 − 0.3×0.64 0.1024
3 2.128 0.256 2.128 − 0.3×0.256 0.0164
4 2.051 0.102 … 0.0026
Pattern: gradient shrinks by 60% every step (×0.40 each time). With α = 0.3, each gradient is (1 − 2α) = 0.4× the previous one. This isn't coincidence — it's a mathematical consequence of moving 60% closer to the minimum each step. The steps self-regulate: no explicit slowdown needed, the algorithm decelerates naturally.

From Toy to Real

What this looks like on an actual prediction problem

Everything so far used a made-up curve L(w) = (w−2)². Here's what gradient descent looks like on a real task: predicting house prices.

Say your model is price = w × area. You have three houses:

area (ft²) 1000 · 1500 · 2000
actual price ($k) 200 · 280 · 380

Start with a wild guess: w = 0.05 (i.e., $50 per ft²). The model predicts 50, 75, and 100 — way off. The loss (mean squared error across all 3 houses) is enormous.

Now gradient descent kicks in. CS229 shows the gradient of this loss with respect to w is:

dL/dw = −(1/n) · Σ (actual − predicted) · area
n number of training examples (3 here) actual − predicted the error on each house — how far off your guess was area the input feature — how much each house's area contributed to the error

At w = 0.05, errors are huge (150, 205, 280) so the gradient is large — a big corrective step. After a few steps, predictions tighten, errors shrink, gradient shrinks, steps get smaller. The same self-regulating pattern you saw in the toy example. Same algorithm. Real problem.

The only difference between the toy curve and a real ML model is that real models have thousands of parameters — not one w but thousands of ws. Gradient descent runs the same update rule on all of them simultaneously. Computing all those gradients at once efficiently — that's what backpropagation does. Which is exactly the next chapter.

Common Misconceptions

Five wrong beliefs — and why they're connected

These aren't five random mistakes. They form a chain — each one leads to the next. Trace the chain and they all become impossible to forget.

  • ① The gradient tells you where to go.
    The gradient points toward steepest increase in loss. You move in the exact opposite direction. This is the root misconception — get it wrong and every intuition downstream is also wrong. The gradient is a slope reading, not a compass to the answer.
  • ② So just follow the negative gradient long enough and you'll reach the best solution.
    No — the gradient is local information only. Following it step by step can land you in a local minimum, a saddle point, or keep you oscillating if α is too large. There is no guarantee of reaching the global minimum. The algorithm is blind to global structure.
  • ③ Then the real danger must be getting stuck in a bad local minimum.
    This is the famous misconception the DL Book specifically corrects. In high-dimensional spaces, local minima are rare — saddle points are exponentially more common. And most local minima in large networks have acceptably low cost. The real trap is saddle points: gradient ≈ 0, so GD stalls, but the model is nowhere near a good solution. Momentum-based optimizers (Adam) exist precisely to escape these.
  • ④ So use a small learning rate to be safe and avoid all these traps.
    Both extremes fail — just differently. Too large → cost oscillates, may diverge. Too small → training stalls at high cost, and may never escape saddle points at all. The DL Book notes the optimal initial rate is typically higher than whatever looks best in the first 100 iterations. Starting small and staying small is a common mistake, not a safe strategy.
  • ⑤ At least gradient descent and backpropagation are the same thing.
    Completely different algorithms that always run together, which is why they're confused. Backpropagation computes the gradients of the loss with respect to every weight — efficiently, in one backward pass through the network using the chain rule. Gradient descent then uses those gradients to update the weights. Backprop is the measuring instrument. GD is the hand that turns the dial.

Summary

Five things to lock in

  • Update rule: w := w − α · (dL/dw) — move opposite to gradient, scaled by α. Step size is automatically proportional to error magnitude.
  • The gradient is local information only — slope at your current point, nothing about global structure. No guarantee of finding the global minimum.
  • Zero gradient means a critical point — which could be a local minimum, maximum, or saddle point. GD cannot tell them apart.
  • In high-dimensional nets, saddle points are the real obstacle — not local minima. Most local minima have acceptably low cost.
  • GD minimizes training loss only. The practical stopping criterion is validation performance, not gradient size — optimization and generalization are separate.
One question this chapter leaves unanswered: Gradient descent needs the gradient of the loss with respect to every parameter — potentially millions of them in a real model. Computing each one individually would be impossibly slow. So how does every modern ML framework compute millions of gradients in a single pass?

That's backpropagation. And it's the next chapter.

Test Your Understanding

10 Questions

0 / 10 revealed

01 You double the learning rate. The model starts overshooting the minimum every step. What should you try instead of just halving it back? ▼
Use a learning rate schedule (decay). Start with the higher rate for faster early progress, then reduce it as training approaches convergence. Simply halving returns you to the original problem of slow convergence. Schedules like cosine annealing or step decay give you the best of both. DL Book §8.3.1 recommends ε_τ ≈ 1% of the initial rate after a few hundred passes through the data.
02 If the gradient at the current point is zero, gradient descent stops moving. Does that mean the model has found the best possible solution? ▼
Not necessarily. A gradient of zero means you're at a critical point — which could be a local minimum, local maximum, or saddle point (DL Book §4.3). In high-dimensional neural networks, saddle points are exponentially more common than local minima (DL Book §8.2.3). Momentum-based optimizers like Adam are used specifically because they can push through these flat regions.
03 Starting from w = 3.0, loss L(w) = (w−2)², learning rate α = 0.4. What is w after exactly one step? ▼
w = 2.2. Gradient at w = 3: dL/dw = 2(3−2) = 2. Update: w = 3 − 0.4 × 2 = 3 − 0.8 = 2.2. Loss drops from (3−2)² = 1 to (2.2−2)² = 0.04 — a 96% reduction in one step.
04 Two models train on identical data. Model A uses batch gradient descent. Model B uses stochastic gradient descent (one sample per step). Model B converges first. Why? ▼
Model B takes far more steps per unit of time. DL Book §8.1.3: returns on larger batches are sublinear — going from 100 to 10,000 samples reduces gradient error by only 10×, while costing 100× more compute. SGD's noisy gradient is cheap to compute, so you take many more steps per second. CS229 also notes SGD often gets θ close to the minimum much faster than batch GD, though it may never fully converge — it oscillates around the minimum.
05 The loss is not decreasing after many steps, but the gradient is not zero either. Name two possible causes. ▼
1. Learning rate too large — each step overshoots, loss oscillates. DL Book §8.3.1: this shows up as "violent oscillations" in the training curve. 2. Ill-conditioning — DL Book §8.2.1: when the Hessian has a poor condition number, the gradient is large in some directions and tiny in others; a learning rate that avoids overshooting in steep directions moves painfully slowly in flat ones. Both look like "stuck" but need opposite fixes.
06 Is it possible for gradient descent to increase the loss on a given step? When? ▼
Yes — when α is too large. DL Book §4.3 shows this with the Taylor expansion: the update adds a curvature correction term (1/2)ε²g^T·H·g to the cost. When this term exceeds the linear decrease ε·g^T·g, the step moves uphill. The gradient guarantees a downhill direction only for infinitesimally small steps. For finite steps, overshooting is always possible — and at high enough α, each step makes things worse (divergence).
07 You normalize all input features to [0,1] before training. Why does this specifically help gradient descent? ▼
It makes the loss landscape more symmetric so one learning rate works for all parameters. DL Book §8.2.1: when the Hessian is ill-conditioned (very different curvatures in different directions), gradient descent performs poorly. A feature with range [0, 10000] produces gradients orders of magnitude larger than one with range [0, 1]. Normalization brings curvatures into similar ranges — the loss surface goes from a stretched ellipse to something closer to a circle, and a single α works reasonably across all dimensions.
08 Starting at w = 0.5, L(w) = (w−2)², α = 0.3. After one step, is the gradient larger or smaller in magnitude than before? ▼
Smaller. At w = 0.5: gradient = 2(0.5−2) = −3. Step: w = 0.5 − 0.3×(−3) = 1.4. At w = 1.4: gradient = 2(1.4−2) = −1.2. Magnitude drops from 3 to 1.2. This is the proportionality property CS229 highlights: gradient magnitude = 2 × distance from minimum. Each step closes the distance, so each subsequent gradient is smaller — the updates naturally decelerate as you converge.
09 Backpropagation is often called "the algorithm that trains neural networks." Is this accurate? ▼
Only half accurate. Backpropagation computes the gradients of the loss with respect to every weight using the chain rule — efficiently, in one backward pass. But it doesn't update anything. Gradient descent uses those gradients to move the weights. Calling backprop "the training algorithm" is like saying "measuring the slope" climbs a mountain. Both are required; neither alone does the job.
10 A model achieves 99% training accuracy but 60% test accuracy. Is gradient descent working correctly? ▼
Yes — gradient descent is working exactly as designed. The problem is overfitting, not optimization. DL Book Ch.8: training algorithms minimize a surrogate loss and halt when a validation-based criterion fires — not when the gradient reaches zero. GD minimized the training loss (99% accuracy). The gap with test accuracy is a generalization problem, handled by regularization, dropout, and early stopping — not by the optimizer. This is why the DL Book treats optimization and generalization as separate concerns.
